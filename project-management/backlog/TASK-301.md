# TASK-301: Contact page missing form component — form never embedded in Canvas page tree

**Type:** Bug
**Priority:** P1 — High
**Severity:** High
**Found:** Sprint 18 QA — Visual Review (2026-03-20)

## Description

The contact page is generated with only a hero banner and a text-media section that says "Fill out the form below" — but no actual form is rendered on the page. The system generates contact form field definitions (`forms.contact.fields` in the blueprint) and imports them as a Drupal webform entity, but the form is **never composed into the Canvas page component tree**.

The Space DS manifest includes form components (`space-form`, `space-input`, `space-textarea`, `space-select`) that are purpose-built for this, but the component tree builder and page generation logic never reference them.

Additionally, the contact page design rules (`page-design-rules.ts:345`) hard-cap sections at `[2, 3]`, which combined with no form component means the contact page is just a hero + a paragraph — effectively a dead-end page.

## Root Cause Analysis

1. **`generator.ts:128-145`** — Fallback contact page only includes `space-hero-banner-style-03` + `space-text-media-default`. No form section.
2. **`component-tree-builder.ts`** — Never references `space-form`, `space-input`, `space-textarea`, or `space-select`.
3. **`page-design-rules.ts:345`** — Contact page `sectionCountRange: [2, 3]` leaves no room for a form section alongside hero + contact info.
4. **Form generation is disconnected** — `buildFormPrompt()` generates field definitions that are saved as `forms.contact` in the blueprint, but this data is only consumed by the Drupal import service (`BlueprintImportService.php`) to create a standalone webform entity. There's no bridge connecting the generated form fields to the Canvas page layout.

## Technical Notes

- The `space-form` component has a `content` slot for child components (inputs, textareas, selects)
- Each `space-input` needs: `type`, `label`, `placeholder`, `required` props
- Each `space-textarea` needs: `label`, `placeholder`, `required`, `rows` props
- Each `space-select` needs options array mapping
- The form section should use the fields already generated by `buildFormPrompt()` — don't duplicate AI calls
- Consider whether the Drupal webform import should also reference the Canvas form block, or if the Canvas form is the primary rendering method
