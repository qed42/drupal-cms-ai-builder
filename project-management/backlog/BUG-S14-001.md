# BUG-S14-001: props_json contains malformed JSON causing parse failure during generation

**Task:** TASK-269 (Generate Pipeline)
**Severity:** High
**Status:** Fixed
**Sprint:** 14

## Steps to Reproduce
1. Start the onboarding flow and complete all steps
2. Trigger blueprint generation (Research → Plan → Generate)
3. During the Generate phase, the AI returns `props_json` as a JSON-encoded string
4. `JSON.parse()` fails on the AI's response

## Error Logs (from `docker logs space-ai-platform`)
```

    at JSON.parse (<anonymous>)
> 96 |           props: JSON.parse(sanitized) as Record<string, unknown>,

Blueprint generation failed: SyntaxError: Unexpected non-whitespace character after JSON at position 1364 (line 1 column 1365)
    at JSON.parse (<anonymous>)
> 96 |           props: JSON.parse(sanitized) as Record<string, unknown>,
```

## Root Cause
The AI sometimes returns malformed `props_json` strings:
- Unescaped quotes inside string values (e.g., `{"title": "Joe's Plumbing"}` — the `'` breaks nothing, but `{"title": "He said "hello""}` does)
- Truncated JSON when the AI hits the token limit mid-string
- Trailing content after the JSON object closes

The current sanitizer only strips control characters (`[\x00-\x1F\x7F]`) but doesn't handle:
1. Truncated JSON (no closing `}`)
2. Unescaped double quotes inside string values
3. Trailing garbage after valid JSON

## Recommended Fix
In `generate.ts` line 96, wrap `JSON.parse` in a try/catch with recovery:
1. Try parsing as-is
2. On failure: attempt to extract the first valid JSON object using regex or a streaming parser
3. On second failure: use an empty props object `{}` and log the raw string for debugging
4. Consider adding `props_json` format validation to the Zod schema (e.g., a `.refine()` that checks `JSON.parse` succeeds)
