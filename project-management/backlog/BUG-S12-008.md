# BUG-S12-008: OpenAI rejects `propertyNames` in structured output schema

**Sprint:** 12
**Severity:** High
**Status:** Fixed

## Description
Generate phase crashes with: `400 Invalid schema for response_format 'response': In context=('properties', 'sections', 'items', 'properties', 'props'), 'propertyNames' is not permitted.`

## Root Cause
`z.record(z.string(), z.unknown())` in `PageSectionSchema.props` generates JSON Schema with `propertyNames` keyword, which OpenAI's strict structured output mode rejects.

## Fix
Changed `props` to `props_json: z.string()` (JSON-encoded string). Updated prompt to instruct AI to return stringified props. Added `JSON.parse()` transform in generate phase to convert back to props object.

## Files Changed
- `src/lib/pipeline/schemas.ts`
- `src/lib/ai/prompts/page-generation.ts`
- `src/lib/pipeline/phases/generate.ts`
