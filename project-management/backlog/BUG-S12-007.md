# BUG-S12-007: Continue Setup starts from scratch instead of resuming

**Sprint:** 12
**Severity:** High
**Status:** Fixed

## Description
Clicking "Continue Setup" on the dashboard for a partially completed onboarding project navigates to `/onboarding/start` with no data, instead of resuming at the step where the user left off with previously filled values.

## Root Cause
1. `SiteCard` hardcoded navigation to `/onboarding/start` without siteId
2. Resume/Save APIs found sessions by `userId` only (no siteId scoping), breaking multi-site
3. When blueprint generation failed, `OnboardingSession.completed` stayed `true` while site status reverted to `onboarding` — resume API filtered for `completed: false` and found nothing
4. No `siteId` in onboarding URLs meant no way to identify which project was being set up

## Fix
- Created `useOnboarding` hook centralizing siteId-aware resume/save/navigation
- Resume API falls back to completed sessions when siteId is explicitly provided
- Save API re-opens completed sessions when user saves new data
- Generate-blueprint failure handler now reverts `completed: false`
- All onboarding URLs include `?siteId={id}`
- SiteCard fetches saved step before navigating
- StepLayout preserves siteId in Back button navigation
