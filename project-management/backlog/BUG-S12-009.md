# BUG-S12-009: Generate phase crashes on control characters in props_json

**Sprint:** 12
**Severity:** Medium
**Status:** Fixed

## Description
After fixing BUG-S12-008, generate phase crashes with: `Bad control character in string literal in JSON at position 178`

## Root Cause
AI returns `props_json` strings containing unescaped control characters (literal newlines, tabs) inside multi-paragraph description values. `JSON.parse()` rejects these.

## Fix
Added sanitization step: `s.props_json.replace(/[\x00-\x1F\x7F]/g, " ")` before `JSON.parse()` to strip control characters.

## Files Changed
- `src/lib/pipeline/phases/generate.ts`
