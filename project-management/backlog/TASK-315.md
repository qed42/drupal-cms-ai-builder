# TASK-315: Migrate BrandTokenService to write Space DS v2 theme settings

**Story:** Space DS v2 Migration
**Priority:** P0
**Estimated Effort:** M
**Milestone:** Space DS v2 Migration

## Description

The current `BrandTokenService` generates raw CSS with `--space-color-*` variables and writes to `public://brand/brand-tokens.css`. The new Space DS v2 theme manages brand colors natively via `space_ds.settings` config and generates CSS variables (`--sds-*`) at runtime in `space_ds_preprocess_page()`.

Rewrite the service to write theme settings instead of generating CSS.

## Technical Approach

### 1. Replace CSS generation with config writes

```php
// OLD: Generate CSS file
$css = $this->generateRootBlock($colors, $fonts);
$this->fileSystem->saveData($css, 'public://brand/brand-tokens.css');

// NEW: Write to theme settings config
$config = $this->configFactory->getEditable('space_ds.settings');
$config->set('base_brand_color', $tokens['colors']['brand'] ?? '#cbddbb');
$config->set('accent_color_primary', $tokens['colors']['primary'] ?? '#658844');
// ... map all color tokens
$config->save();
```

### 2. Color token mapping

| Blueprint Token Key | Theme Setting Key | Fallback |
|---------------------|------------------|----------|
| `colors.brand` / `colors.base` | `base_brand_color` | #cbddbb |
| `colors.primary` | `accent_color_primary` | #658844 |
| `colors.secondary` | `accent_color_secondary` | #ffffff |
| `colors.text` / `colors.neutral` | `neutral_color` | #192211 |
| `colors.heading` | `heading_color` | #2f3941 |
| `colors.border` | `border_color` | #ebe6e7 |
| `colors.gray` | `gray_color` | #efefef |
| `colors.success` | `success_color` | #dcfce7 |
| `colors.danger` | `danger_color` | #ffe2e2 |
| `colors.warning` | `warning_color` | #fef9c2 |
| `colors.info` | `info_color` | #dbeafe |
| `colors.background_1..10` | `background_color_1..10` | Theme defaults |

### 3. Typography mapping

| Blueprint Token Key | Theme Setting Key | Fallback |
|---------------------|------------------|----------|
| `fonts.body` / `fonts.primary` | `font_family` | Geist |
| `fonts.size` / `fonts.base_size` | `base_font_size` | 18 |

Note: Theme only supports 3 font families (Geist, sans-serif, serif). Map closest match.

### 4. Keep logo handling
The `applyLogo()` method remains valid â€” it writes to `{theme}.settings.logo.*` which works with any theme.

### 5. Remove CSS generation methods
- Delete `generateRootBlock()`
- Delete `generateGoogleFontsImport()` (theme handles its own fonts)
- Delete `generateFontFaceDeclarations()` (custom fonts need separate approach)
- Delete `CSS_OUTPUT_URI` constant (no more CSS file)

### 6. Update Drush command
`ApplyBrandCommands.php` may need updating if it references the CSS file path.
