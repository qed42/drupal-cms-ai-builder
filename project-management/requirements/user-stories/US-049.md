# US-049: Per-Page Regeneration

**Epic:** Content Review & Editing
**Priority:** P1
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-304

## User Story
As a business owner, I want to regenerate an entire page while keeping other pages unchanged, so that I can get a fresh take on a page that doesn't meet my expectations.

## Acceptance Criteria
- [ ] Given the review page, when the user clicks "Regenerate Page" on any page, then the entire page is regenerated
- [ ] Given page regeneration, when triggered, then it uses the same research brief and content plan to ensure consistency with the rest of the site
- [ ] Given page regeneration, when it completes, then only the targeted page is replaced — all other pages remain unchanged
- [ ] Given page regeneration, when it completes, then the previous page version is preserved for undo

## Dependencies
- US-048 (Per-Section AI Regeneration)

## Technical Notes
- Similar to per-section regeneration but regenerates all sections on the page in a single AI call
- API endpoint: `POST /api/blueprint/{id}/regenerate-page` with `{ pageIndex, guidance? }`
- Can reuse the per-page generation logic from US-041
