# US-108: Inject Design Tokens into Code Component Generation Prompt

**Epic:** Design Quality & Consistency
**Priority:** P0
**Milestone:** M27 — Design Rules Engine

## User Story
As the AI generation pipeline, I need to inject resolved design tokens into the code component system prompt, so that every generated section uses the same visual vocabulary.

## Acceptance Criteria
- [ ] Given `ENABLE_DESIGN_RULES=true` and `generationMode=code_components`, when the generate-code-components phase runs, then the resolved design tokens fragment is passed to every `generateCodeComponent()` call
- [ ] Given the code component generation prompt receives a tokens fragment, when the AI generates a section, then the AI is instructed to use the EXACT Tailwind classes from tokens for container, typography, cards, and buttons
- [ ] Given background alternation tokens, when multiple sections are generated, then the prompt instructs the AI to follow the alternation pattern (section index is passed to enable this)
- [ ] Given the feature flag is disabled, when code components are generated, then generation proceeds without tokens (no regression)
- [ ] Integration test verifies the full path: onboarding data → rules resolution → prompt injection → generation call

## Dependencies
- US-107 (Design Tokens)
- US-102 (Pipeline Mode Branching)

## Technical Notes
- `generate-code-components.ts` already resolves rules and passes `designRulesFragment` — verify tokens are included in the fragment
- The section index should be passed to enable background alternation awareness
- Consider adding section count context so the AI knows total page structure
