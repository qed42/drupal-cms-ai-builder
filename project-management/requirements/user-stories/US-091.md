# US-091: Separate Image Resolution from Content Generation

**Epic:** Image Pipeline
**Priority:** P0
**Milestone:** M22 — User-Uploaded Images (bug fix)

## User Story
As a site builder, I want the image resolution pipeline to run as a separate phase after content generation, so that my uploaded photos can be matched to the correct sections instead of being ignored.

## Acceptance Criteria
- [ ] Generate phase produces sections with empty image props (no Pexels calls)
- [ ] Enhance phase is the sole owner of image resolution (user images + Pexels fallback)
- [ ] User-uploaded images matched to sections when score >= 0.25
- [ ] Pexels stock images fill remaining empty slots
- [ ] Component trees built once (in Enhance, after images injected)
- [ ] No behavior change for sites without user images
- [ ] `_meta.imageSource` correctly set on every image-bearing section

## Dependencies
- None (refactor of existing code)

## Technical Notes
- See `architecture-image-pipeline-refactor.md` for full design
- Primary change: remove `resolveImagesForSections()` from `generate.ts`
- Enhance phase already has correct logic — currently short-circuited by pre-filled slots
