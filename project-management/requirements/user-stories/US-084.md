# US-084: Drupal Media Entity Creation for User Images

**Epic:** User-Uploaded Images
**Priority:** P1
**Milestone:** M22 — User-Uploaded Images

## User Story
As a site owner, I want all my uploaded images to become proper Drupal Media entities after provisioning, so that I can manage, reuse, and assign them via Drupal's Media Library — even images the AI didn't place automatically.

## Acceptance Criteria
- [ ] Given a provisioned site, when provisioning step 08.3 runs, then ALL user-uploaded images (used + unused in blueprint) are copied to the Drupal files directory at `sites/{domain}/files/user-images/`
- [ ] Given each user image, when a Media entity is created, then it has: file entity, media entity (type: image), alt text from AI description, and original filename as name
- [ ] Given the media creation, when complete, then a media ID map `{ imageId → drupalMediaId }` is returned
- [ ] Given blueprint sections that reference user images, when media entities are created, then blueprint paths are rewritten from `/uploads/...` to Drupal media entity references with `media_id`
- [ ] Given unused uploaded images, when provisioning completes, then they are browsable in the Drupal Media Library for the user to assign manually later
- [ ] Given the Drush batch command, when invoked with a manifest JSON, then it processes all images in a single Drush invocation (not one invocation per image)

## Dependencies
- US-083 (Image Matching — determines which images are used in the blueprint)

## Technical Notes
- New provisioning step: `08.3-create-media-entities.ts` (after image copy, before blueprint import)
- New Drush command: `ai-site-builder:import-media --manifest=/tmp/media-manifest.json`
- Manifest format: `[{ file, alt, name, bundle }]` — returns `[{ image_id, media_id, fid }]`
- Drupal Media type: `image` with `field_media_image → file` reference
- Expected latency: ~500ms per image via batch manifest
- Blueprint rewrite: `{ src: "/uploads/..." }` → `{ media_id: 42, src: "/sites/{domain}/files/user-images/..." }`
- See `architecture-user-images.md` Sections 4.5–4.6 for full specification
