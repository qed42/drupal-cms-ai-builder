# US-112: SDC Approximate Rendering

**Epic:** Live Blueprint Preview
**Priority:** P1
**Milestone:** M28 — Live Blueprint Preview

## User Story
As a user who chose the design_system generation mode, I want to see an approximate visual preview of my SDC component sections, so that I have a similar preview experience to code_components users even though exact Drupal rendering is not available in the browser.

## Acceptance Criteria
- [ ] Given a blueprint with design_system generation mode, when the review page loads, then each section is rendered using an approximate HTML renderer that visually resembles the SDC component
- [ ] Given an SDC hero section with heading, subheading, and CTA button props, when rendered in the preview, then the approximate renderer shows a full-width hero layout with the correct text content and a styled button
- [ ] Given an SDC composed section with a flexi grid and card children, when rendered in the preview, then the approximate renderer shows a grid layout with cards in the correct column arrangement
- [ ] Given at least the following SDC component types, when any of them appear in a blueprint, then an approximate renderer exists: hero, section-heading, card, button, container, flexi, accordion, slider, contact-card, image, paragraph
- [ ] Given an SDC component type without a registered approximate renderer, when rendered in the preview, then a generic "Unsupported Component" placeholder is shown with the component name and prop summary
- [ ] Given brand tokens are configured, when SDC approximate renderers are used, then they apply the same CSS custom properties as code component rendering

## Dependencies
- US-111 (Live Preview Frame — the iframe infrastructure)
- US-113 (Brand Token Injection)

## Technical Notes
- Architecture: `architecture-live-blueprint-preview.md` Section 3.5
- New file: `platform-app/src/lib/preview/sdc-renderers.ts` — registry of approximate renderers
- Renderers are lightweight React functional components that accept `{ props: Record<string, unknown> }`
- SDC approximate renderers are explicitly documented as "approximate" — they are not pixel-perfect replicas of the Drupal-rendered SDC components
- The same iframe sandbox and brand token injection from US-111 and US-113 are reused
- Composed sections (Type B) need a FlexiApproximate renderer that handles column layout with children
