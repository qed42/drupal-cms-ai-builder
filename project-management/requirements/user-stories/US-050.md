# US-050: Page Add/Remove from Review

**Epic:** Content Review & Editing
**Priority:** P0
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-305

## User Story
As a business owner, I want to add new pages or remove existing pages from the review interface, so that I can customize my site structure after seeing the generated content.

## Acceptance Criteria
- [ ] Given the review page, when the user clicks "Add Page", then they can enter a page title and description for AI generation
- [ ] Given a new page is added, when the AI generates it, then it uses the existing research brief and content plan for consistency
- [ ] Given the review page, when the user clicks "Remove Page" on an existing page, then the page is removed from the blueprint after confirmation
- [ ] Given page additions, when the total exceeds 15 pages, then the system prevents adding more and shows the limit
- [ ] Given a removed page, when it is confirmed, then the page is removed from the blueprint JSON

## Dependencies
- US-046 (Markdown Content Preview Page)
- US-048 (Per-Section AI Regeneration)

## Technical Notes
- Added pages use the same per-page generation logic as US-041
- Page removal is soft-delete from the blueprint JSON — no database record deletion needed
- Maximum 15 total pages per PRD constraint
