# US-036: Enhanced AI Page Suggestions

**Epic:** Onboarding Enhancements
**Priority:** P0
**Milestone:** M6 — Onboarding Enrichment
**PRD Refs:** FR-106

## User Story
As a business owner, I want to see AI-suggested pages with descriptions of what each page will contain, so that I can make informed decisions about my site structure.

## Acceptance Criteria
- [ ] Given the page suggestion step, when the AI generates suggestions, then it receives the full user idea text (not just detected industry)
- [ ] Given AI page suggestions, when they are displayed, then each suggestion includes both a page title and a 1-sentence description explaining what the page will contain
- [ ] Given page suggestions, when the user views them, then they see descriptions like "Services — Detailed descriptions of your dental procedures with patient benefits and expected outcomes"
- [ ] Given page suggestions with descriptions, when the user selects/deselects pages, then the selections and their descriptions are stored for the content pipeline

## Dependencies
- US-005 (Site Basics Input)
- US-006 (Industry Selection)

## Technical Notes
- This enhances the existing `/api/onboarding/suggest-pages` endpoint
- The current endpoint only returns page titles — needs to return `{ title: string, description: string }[]`
- The description feeds into the Plan and Generate phases as page purpose context
- Consider caching suggestions per industry to reduce AI calls for common industries
