# US-059: Content Evaluation Rubric

**Epic:** Quality Assurance
**Priority:** P0
**Milestone:** M10 — Quality Framework
**PRD Refs:** FR-504

## User Story
As a QA engineer, I want a codified scoring rubric that evaluates generated content across 8 dimensions, so that I can objectively measure and track content quality improvements.

## Acceptance Criteria
- [ ] Given the rubric, when applied, then it scores content across 8 dimensions: content relevance (0-5), content depth (0-5), industry accuracy (0-5), page structure (0-5), SEO quality (0-5), tone consistency (0-5), CTA relevance (0-5), component selection (0-5)
- [ ] Given the scoring, when totaled, then the maximum score is 40 with a pass threshold of 28/40 (70%)
- [ ] Given automated criteria, when evaluated, then word count, keyword presence, and content uniqueness are checked automatically
- [ ] Given subjective criteria, when evaluated, then manual evaluation guidelines are provided for dimensions like tone consistency and content relevance
- [ ] Given the rubric, when codified, then it exists as a JSON schema with both automated and manual evaluation rules

## Technical Notes
- Open Question OQ-7: Should scoring be partially automated via a separate AI call? Recommend starting with manual + automated hybrid, add AI grading later if needed
- Automated checks: word count thresholds, keyword matching, Jaccard similarity for uniqueness
- Manual checks: provide clear rubric descriptions with examples of 1/3/5 scores per dimension
- Consider a simple CLI tool or script that runs automated checks and outputs a partial score
