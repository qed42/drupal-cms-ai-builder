# US-012: Content Type Auto-Creation

**Epic:** AI Website Generation
**Priority:** P0
**Milestone:** M2 — AI Site Generation Engine

## User Story
As a user who completed onboarding, I want the AI to automatically create structured Drupal content types based on my business context, so that my website has a proper content architecture — not just flat pages.

## Acceptance Criteria
- [ ] Given onboarding data includes services, when generation runs, then a "Service" content type is created with fields: title, description, image, CTA
- [ ] Given a Healthcare industry, when generation runs, then a "Provider" content type is created with fields: name, photo, specialization, bio
- [ ] Given any industry, when generation runs, then a "Team Member" content type is created with fields: name, photo, role, bio
- [ ] Given the generated content types, when viewed in Drupal admin, then they follow Drupal entity/field best practices (proper field types, labels, form/display modes)
- [ ] Given content types are created, when generation continues, then sample content entities are created for each type (US-017)

## Dependencies
- US-009 (Dynamic Industry Questions — onboarding data must be complete)

## Technical Notes
- FR-202
- AI Agent determines which content types are needed based on industry + business context
- Content types created programmatically via Drupal Entity API
- Must handle idempotency — if generation is re-run, don't create duplicate content types
- Reference: Appendix B in PRD for industry-specific content types
