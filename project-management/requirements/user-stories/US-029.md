# US-029: Draft Mode & Content Preview

**Epic:** Publishing & Deployment
**Priority:** P0
**Milestone:** M5 — Publishing & Subscription

## User Story
As a user reviewing my generated site, I want all content to start in draft mode with a preview capability, so that I can review and edit before anything goes live.

## Acceptance Criteria
- [ ] Given the AI generates content, when creation completes, then all pages and content entities are in "draft" (unpublished) status
- [ ] Given draft content, when the user views the site in Canvas, then they see a full preview as if the site were live
- [ ] Given a draft site, when an anonymous visitor tries to access it, then they see nothing (no public access until published)
- [ ] Given draft pages, when viewed in the editor, then they are clearly marked as "Draft" with a visual indicator

## Dependencies
- US-013 (Page Generation)

## Technical Notes
- FR-502
- Drupal's content moderation / publishing workflow
- Preview mode via Canvas or Drupal's built-in preview
- Anonymous access blocked until explicit publish action
