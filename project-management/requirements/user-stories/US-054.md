# US-054: Step-Level Provisioning Progress with Timing

**Epic:** Provisioning UX
**Priority:** P0
**Milestone:** M9 — Provisioning Hardening
**PRD Refs:** FR-401, FR-403

## User Story
As a business owner, I want to see each provisioning step's status and timing in real time, so that I understand exactly what is happening while my site is being built.

## Acceptance Criteria
- [ ] Given provisioning starts, when the user views the progress page, then all 11 provisioning steps are displayed with individual status indicators: completed (with elapsed time), in progress (with spinner), pending, or failed (with error summary)
- [ ] Given step names, when displayed, then they use user-friendly labels (e.g., "Setting up your database" not "create-database")
- [ ] Given progress updates, when delivered, then they are sent via the existing polling mechanism (`/api/provision/status`) with a new `currentStep` field
- [ ] Given each step, when it runs, then it records start time, end time, and duration
- [ ] Given step timings, when displayed, then each completed step shows its elapsed time (e.g., "Installing Drupal CMS... 42s")
- [ ] Given step timings, when all steps complete, then aggregate timing is logged for performance monitoring
- [ ] Given progress indicators, when they update, then ARIA live regions announce status changes for screen readers (NFR-18)

## Dependencies
- US-052 (Approve & Provision Flow)

## Technical Notes
- Enhances existing `/api/provision/status` endpoint to include step-level detail
- The provisioning engine already has 11 steps — need to add timing instrumentation to each
- Step-friendly names mapping: `{ 'create-database': 'Setting up your database', 'install-drupal': 'Installing Drupal CMS', ... }`
- Consider storing step timings in the site record for historical performance analysis
