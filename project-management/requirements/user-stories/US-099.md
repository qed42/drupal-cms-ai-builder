# US-099: Code Component Type System & Config Builder

**Epic:** Code Component Generation
**Priority:** P0
**Milestone:** M26 — Code Component Generation

## User Story
As a platform developer, I want a strongly-typed interface for Code Component generation and a config YAML builder that produces valid Canvas config entities, so that generated components integrate seamlessly with Drupal Canvas.

## Acceptance Criteria
- [ ] Given the `CodeComponentGenerator` interface, when implemented, then it defines `generate()`, `validate()`, and `buildConfig()` methods with proper TypeScript types
- [ ] Given a `CodeComponentOutput` (machineName, name, jsx, css, props, slots), when passed to `buildConfigYaml()`, then it produces valid `canvas.js_component.[machineName].yml` YAML matching Canvas's config entity schema
- [ ] Given generated YAML, when it includes props, then each prop has the correct Canvas type mapping (string, formatted_text, boolean, integer, number, link, image, video, list:text, list:integer)
- [ ] Given a Code Component output, when wrapped as a Canvas tree item, then the `component_id` uses `js.[machineName]` format with proper UUID, parent_uuid, slot, and inputs structure
- [ ] Given the config builder, when generating entity IDs, then machine names are valid Drupal identifiers (lowercase, underscores, no special characters)
- [ ] Given unit tests, when all tests run, then 100% of config builder functions are covered

## Dependencies
- None
