# US-043: Structured Output Enforcement & Retries

**Epic:** AI Content Pipeline
**Priority:** P0
**Milestone:** M7 — AI Content Pipeline
**PRD Refs:** FR-207

## User Story
As a platform developer, I want all AI calls to use structured output with validation, so that the pipeline never proceeds with invalid or malformed data.

## Acceptance Criteria
- [ ] Given any AI call in the pipeline, when it is made, then it uses structured output (JSON schema or function calling) to guarantee valid responses
- [ ] Given AI output, when it is received, then the system validates it against the corresponding TypeScript interface before proceeding
- [ ] Given invalid AI output, when validation fails, then the system retries (max 2 retries) with the validation error appended to the prompt
- [ ] Given all retries exhausted, when validation still fails, then the phase fails with a clear error message
- [ ] Given structured output enforcement, when applied, then it works identically across OpenAI and Anthropic providers

## Dependencies
- US-038 (AI Provider Abstraction Layer)

## Technical Notes
- OpenAI supports `response_format: { type: "json_schema" }` and function calling
- Anthropic supports structured output via tool use
- Use Zod schemas for TypeScript validation — they can generate JSON Schema for the AI call and validate the response
- Consider a wrapper: `generateValidatedJSON<T>(prompt, zodSchema): Promise<T>` that handles retry logic
- Log validation failures for prompt improvement iteration
