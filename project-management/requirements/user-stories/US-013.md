# US-013: Page Generation with SDC Components

**Epic:** AI Website Generation
**Priority:** P0
**Milestone:** M2 — AI Site Generation Engine

## User Story
As a user who completed onboarding, I want the AI to generate complete pages using the Space design system components, so that I get a professionally designed website without any manual layout work.

## Acceptance Criteria
- [ ] Given onboarding is complete, when the user clicks "Generate My Website", then at minimum Home, About, Services, and Contact pages are created
- [ ] Given a specific industry, when pages are generated, then industry-specific pages are included (e.g., "Doctors" for Healthcare, "Listings" for Real Estate)
- [ ] Given generated pages, when viewed, then each page is composed exclusively of Space SDC components (no custom HTML)
- [ ] Given generated pages, when viewed, then layouts follow the design system's layout rules and spacing conventions
- [ ] Given generated pages, when inspected, then they use proper Drupal Canvas structures (sections, components) editable via Canvas

## Dependencies
- US-012 (Content Type Auto-Creation)
- Space design system SDC components must be available

## Technical Notes
- FR-201, FR-205, FR-207
- Page Builder Agent maps content needs to available SDC components
- Component whitelist enforcement — agent cannot reference components outside the Space library
- Pages stored as Drupal nodes with Canvas layout data
- Need a component catalog/manifest that the AI agent can reference
