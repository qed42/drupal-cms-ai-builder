# US-065: AI Strategy Preview on Review Settings Page

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 2)

## User Story
As a user about to generate my website, I want to see the AI's strategy for my site before I click Generate, so that I can verify it understood my goals and course-correct if needed.

## Acceptance Criteria
- [ ] Given a user navigates to the review-settings page, when the page loads, then a Research preview is triggered automatically and a loading skeleton is shown
- [ ] Given the Research preview completes, when the user views the "AI Strategy Preview" panel, then it shows: page structure with section types per page, tone guidance with 1-2 example sentences, SEO keyword distribution across pages, and competitive positioning summary
- [ ] Given the Research preview is loading, when the user views the page, then the existing input summary (name, idea, audience, pages, colors, fonts, tone) is visible immediately — the preview panel shows a skeleton below it
- [ ] Given the Research preview completes in under 20 seconds, when the result is cached, then it is re-used as the Research phase input during actual generation (no duplicate AI call)
- [ ] Given the Research preview fails, when the error occurs, then the page degrades gracefully to current behavior with a "Preview unavailable" note and the Generate button remains available
- [ ] Given the user goes back from review-settings and changes inputs, when they return to review-settings, then the cached preview is invalidated and re-fetched
- [ ] Given the Generate button is displayed, when the user views it, then it is always clickable regardless of preview status — the preview is informational, not blocking
- [ ] Given the competitive positioning section, when it renders, then it does NOT show real competitor names — uses generic phrasing like "typical [industry] websites"
- [ ] Given the user is on desktop, when the panel loads, then it is expanded by default
- [ ] Given the user is on mobile, when the panel loads, then it is collapsed by default with a tap-to-expand affordance

## Dependencies
- US-066 (Research Preview API) must be implemented first

## Technical Notes
- This is the highest-impact feature — it's the last chance to build trust before the irreversible generation step.
- The Research phase (~10-15s) produces exactly the data needed: ResearchBrief with industry analysis, audience insights, tone guidance, SEO keywords, competitive positioning.
- Caching strategy: store the preview result in the OnboardingSession or a new `research_previews` field. On generation, check if a valid preview exists and skip the Research phase.
- Consider running the Research preview eagerly after the Audience step (background) to eliminate wait time on review-settings. This is an optimization — not required for v1.
