# US-103: Image Enhancement for Code Components

**Epic:** Code Component Generation
**Priority:** P1
**Milestone:** M26 — Code Component Generation

## User Story
As a user viewing my generated site, I want placeholder images in code component sections replaced with relevant stock photos, so that the site looks realistic and professional before I add my own images.

## Acceptance Criteria
- [ ] Given generated JSX containing `<img>` tags with `/placeholder/{width}x{height}` src attributes, when the Enhance phase runs, then each placeholder is replaced with a contextually relevant stock image URL
- [ ] Given an image placeholder, when the alt text or surrounding content provides context (e.g., "modern dental office"), then the stock photo search uses that context for better relevance
- [ ] Given the Enhance phase for code components, when images are replaced, then the original placeholder dimensions are preserved in the stock image request
- [ ] Given JSX with background image references in Tailwind classes, when enhancement runs, then background image URLs are also detected and replaced
- [ ] Given a stock photo API failure, when an image cannot be resolved, then the placeholder URL is retained (graceful degradation) rather than breaking the component

## Dependencies
- US-101 (Designer Agent generates placeholders)
- US-102 (pipeline integration)

## Technical Notes
- Architecture: `architecture-code-components.md` Section 6
- Different from SDC image enhancement (which matches props) — this parses JSX for `<img>` tags and Tailwind `bg-[url()]` patterns
- Uses the existing stock photo API (Pexels/Unsplash) already integrated for SDC mode
