# US-096: Persistent Header Across Onboarding Journey

**Epic:** UX Polish
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a user navigating through the onboarding steps, I want to see the site header consistently on every page, so that I can always access navigation and don't feel lost when I leave the dashboard.

## Acceptance Criteria
- [ ] The app header (logo, nav, user menu) is visible on all onboarding step pages
- [ ] The header is visible on the progress page, review page, and review-settings page
- [ ] The header does not duplicate if the page already has its own header section
- [ ] Header styling is consistent with the dashboard header

## Dependencies
- None

## Technical Notes
- The onboarding layout (`app/onboarding/layout.tsx`) likely omits the header — add it there
- Ensure the header doesn't conflict with the ProgressStepper or StepLayout spacing
- Check if the review page has its own header that needs to be reconciled
