# US-115: Section Click-to-Edit

**Epic:** Live Blueprint Preview
**Priority:** P0
**Milestone:** M28 — Live Blueprint Preview

## User Story
As a user reviewing my generated website preview, I want to click on any section in the live preview to open an edit panel where I can modify its content and trigger regeneration, so that I can refine my site without leaving the visual preview context.

## Acceptance Criteria
- [ ] Given the live preview is rendered, when the user hovers over a section, then the section shows a subtle outline/highlight effect indicating it is clickable
- [ ] Given the user clicks a section in the iframe preview, when the click is detected, then a slide-over edit panel opens on the right side showing that section's editable props
- [ ] Given the edit panel is open, when the user modifies a text prop (e.g., heading, description), then the change is reflected in the live preview within 150ms (debounced)
- [ ] Given the edit panel is open, when the user clicks "Regenerate Section", then the regenerate-section API is called and the new section replaces the old one in the live preview
- [ ] Given the edit panel is open, when the user clicks "View Code", then the panel shows the JSX source with syntax highlighting (code_components mode only)
- [ ] Given the user clicks outside the edit panel or clicks a different section, when the interaction occurs, then the current edit panel closes (or switches to the new section)
- [ ] Given the edit panel modifies props, when the changes are made, then they are auto-saved to the server via the existing auto-save mechanism

## Dependencies
- US-111 (Live Preview Frame — the iframe)
- Existing: `useAutoSave` hook, `regenerate-section` API endpoint

## Technical Notes
- Architecture: `architecture-live-blueprint-preview.md` Sections 3.6, 4.2, 4.3
- New file: `platform-app/src/app/onboarding/review/components/EditPanel.tsx`
- Communication between iframe and parent uses `window.postMessage`:
  - iframe → parent: `{ type: "section-click", sectionIndex: number }`
  - parent → iframe: `{ type: "update-props", sectionIndex: number, props: Record<string, unknown> }`
  - parent → iframe: `{ type: "replace-section", sectionIndex: number, jsx?: string, css?: string, props: Record<string, unknown> }`
- Each section in the iframe is wrapped in a `<div data-section-index="N">` with event listeners
- The edit panel reuses prop rendering logic from the existing `SectionView` component in `PagePreview.tsx`
- Prop updates to the iframe are debounced at 150ms using the existing debounce pattern
