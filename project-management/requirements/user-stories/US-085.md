# US-085: Image Provenance Transparency

**Epic:** User-Uploaded Images
**Priority:** P2
**Milestone:** M22 — User-Uploaded Images

## User Story
As a site owner reviewing my generated site, I want to see which images are mine vs. stock photos and why the AI placed each one where it did, so that I can verify the AI's choices and adjust if needed.

## Acceptance Criteria
- [ ] Given the review page, when image provenance data exists, then each section shows whether its image is "Your image" or "Stock photo"
- [ ] Given a user-matched image, when shown on the review page, then it displays the match reason (e.g., "Matched to 'Our Services' — tags: professional, teamwork")
- [ ] Given a Pexels fallback image, when shown on the review page, then it displays "Stock photo — you can replace this with your own image in Drupal"
- [ ] Given the post-generation dashboard, when the user views their site summary, then a callout reads "All {N} images you uploaded are available in your Drupal Media Library"
- [ ] Given any InferenceCard or transparency element showing image info, when it renders, then it uses existing component patterns (no new UI components)

## Dependencies
- US-083 (Image Matching — produces `imageSource` and `imageMatchScore` metadata)
- US-084 (Media Entity Creation — confirms images are in Drupal)

## Technical Notes
- Image provenance data already tracked in `section._meta.imageSource` and `section._meta.imageMatchScore`
- Review page already renders section summaries — extend with image source indicator
- "Archie" branding: use consistent voice (e.g., "Archie matched your photo to this section because...")
- See `architecture-user-images.md` Section 10 for UX transparency guidelines
