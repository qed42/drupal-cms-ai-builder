# US-004: Subscription Activation

**Epic:** User Account & Trial Management
**Priority:** P1
**Milestone:** M5 — Publishing & Subscription

## User Story
As a trial user, I want to subscribe to a paid plan, so that my site stays live after the trial ends.

## Acceptance Criteria
- [ ] Given a trial user, when they click "Subscribe", then they see a payment page with subscription options
- [ ] Given a valid payment, when the subscription is activated, then the trial converts to an active subscription
- [ ] Given an expired trial with offline site, when the user subscribes, then the site comes back online immediately
- [ ] Given an active subscription, when the user logs in, then they see their subscription status and renewal date
- [ ] Given a subscription, when it follows the 1 license = 1 site model, then each subscription is tied to exactly one site

## Dependencies
- US-002 (Free Trial Activation)

## Technical Notes
- Payment gateway integration (Stripe recommended for MVP)
- Subscription entity or Commerce module — evaluate build vs. buy
- Webhook handler for payment events (successful charge, failed charge, cancellation)
- Consider Drupal Commerce vs. lightweight custom implementation
