# US-068: "Why This?" Section Tooltips in Review Editor

**Epic:** Transparent AI Onboarding
**Priority:** P2
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 4)

## User Story
As a user reviewing my generated website, I want to understand why specific sections and content were created, so that I can make informed editing decisions and trust the AI's choices.

## Acceptance Criteria
- [ ] Given any section in the review editor, when the user views it, then a subtle info icon ("?") appears alongside the edit button
- [ ] Given the user clicks/hovers the info icon on a section, when the tooltip opens, then it shows: which user inputs influenced this section (e.g., tone, audience, keywords), which SEO keywords are targeted, and a 1-2 sentence rationale for the content
- [ ] Given the tooltip content, when it is rendered, then it is generated from existing pipeline metadata (ResearchBrief, ContentPlan, review scores) — no additional AI calls
- [ ] Given the user is on desktop, when the tooltip opens, then it renders as a popover positioned relative to the section
- [ ] Given the user is on mobile, when the tooltip opens, then it renders as a bottom sheet
- [ ] Given the tooltip is open, when the user clicks outside it, then it closes without disrupting the editing workflow
- [ ] Given the user has edited a section, when they open the tooltip for that section, then a note appears: "You've customized this section — original AI reasoning may no longer apply"
- [ ] Given an image section, when the tooltip opens, then it shows the search query used (e.g., "Searched: 'family dental office modern'") so the user understands photo selection

## Dependencies
- US-069 (Page-Level Insights Panel)
- Existing review agent data in `blueprint.payload._review`

## Technical Notes
- The tooltip data comes from multiple sources that are already stored:
  - ResearchBrief → tone guidance, audience pain points, keywords
  - ContentPlan → section content briefs, target keywords per page
  - Image intent → search queries stored during generation
- Need to build a mapping layer that connects section component IDs to their originating plan data.
- The `_review` data already exists in the blueprint payload — this just needs a UI to surface it.
- Consider a `<SectionInsight sectionId={id} planData={plan} reviewData={review} />` component.
