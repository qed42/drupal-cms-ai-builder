# US-051: Version Comparison & Original Preservation

**Epic:** Content Review & Editing
**Priority:** P0
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-306, FR-309

## User Story
As a business owner, I want to compare my edited content with the original AI-generated version, so that I can see what changed and revert if needed.

## Acceptance Criteria
- [ ] Given the user makes edits, when the original AI-generated blueprint is created, then it is preserved as a separate record (never overwritten by user edits)
- [ ] Given the review page, when the user clicks "Compare with Original", then they see a diff view showing additions (green) and removals (red)
- [ ] Given the diff view, when displayed, then it compares the current (edited) content with the original AI-generated version per section
- [ ] Given the provisioned site, when it is built, then it uses the edited version (not the original)
- [ ] Given both versions, when the user visits the dashboard, then both the original and edited versions are accessible

## Dependencies
- US-047 (Inline Content Editing)

## Technical Notes
- Store original blueprint as a separate record with a `version: 'original'` flag when the first edit is made
- Diff rendering: use a text diffing library (e.g., `diff` npm package) with line-level or word-level comparison
- The original version is immutable once created — only the "working" version is modified by edits
- Dashboard access to both versions is a UI concern — show "Original" and "Edited" tabs on the site detail page
