# US-109: P1 Design Quality — Heading Hierarchy, Color Patterns & Intra-Section Spacing

**Epic:** Design Quality & Consistency
**Priority:** P1
**Milestone:** M27 — Design Rules Engine

## User Story
As a site builder, I want the AI to enforce consistent heading hierarchy, color usage patterns, and intra-section spacing, so that generated sites have professional vertical rhythm and visual coherence.

## Acceptance Criteria
- [ ] Given design tokens, when a code component is generated, then heading levels follow a strict hierarchy: h1 (hero only) → h2 (section titles) → h3 (card/feature titles) → h4 (sub-headings)
- [ ] Given brand colors are available, when sections are generated, then the prompt specifies when to use `--color-primary` vs `--color-accent` (CTAs use primary, hover states use accent)
- [ ] Given intra-section spacing rules, when a section is generated, then heading→subtitle→content→CTA gaps follow a consistent spacing scale (e.g., mb-4 → mb-6 → mb-8 → mb-12)
- [ ] Given text max-width tokens already exist, when body text is rendered, then it does not stretch beyond the maxWidth token (max-w-2xl)
- [ ] Rules are additive YAML entries in global.yaml (composition and tokens categories)

## Dependencies
- US-107 (Design Tokens)

## Technical Notes
- Heading hierarchy is a composition rule (what semantic levels to use), not a visual token
- Color usage pattern is a visual rule extension
- Intra-section spacing could be a new token or an extension of the sectionSpacing token
- These are P1 — noticeable quality improvement but not broken without them
