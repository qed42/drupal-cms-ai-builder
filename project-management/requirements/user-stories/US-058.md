# US-058: Automated Playwright E2E Tests

**Epic:** Quality Assurance
**Priority:** P0
**Milestone:** M10 — Quality Framework
**PRD Refs:** FR-503

## User Story
As a QA engineer, I want automated Playwright tests that validate the full onboarding-to-review flow, so that content quality regressions are caught automatically.

## Acceptance Criteria
- [ ] Given the test suite, when run, then at least 5 synthetic test cases are automated end-to-end
- [ ] Given each automated test, when it runs, then it verifies: (a) onboarding wizard completes without errors, (b) all three generation phases complete, (c) review page renders with content
- [ ] Given content validation, when checked, then total word count meets minimum threshold (2,000 words)
- [ ] Given per-page validation, when checked, then each page has at least 200 words
- [ ] Given industry validation, when checked, then industry-specific keywords appear in generated content
- [ ] Given placeholder detection, when checked, then no placeholder/lorem ipsum text is found
- [ ] Given CI integration, when tests run, then each test case has a 10-minute timeout

## Dependencies
- US-057 (Synthetic Test Case Suite)
- US-046 (Markdown Content Preview Page)
- US-042 (Pipeline Phase Visibility UI)

## Technical Notes
- Tests run against the full platform stack (Next.js + Drupal + provisioning containers)
- Word count validation: extract text content from the review page DOM
- Keyword matching: maintain per-industry keyword lists in the test fixtures
- Placeholder detection: regex check for common placeholder patterns (lorem ipsum, [placeholder], TBD, etc.)
- Consider test parallelization — each test case is independent
