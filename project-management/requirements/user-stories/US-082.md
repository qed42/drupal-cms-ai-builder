# US-082: AI Image Description & Analysis

**Epic:** User-Uploaded Images
**Priority:** P1
**Milestone:** M22 — User-Uploaded Images

## User Story
As a business owner who uploaded images, I want each image to be automatically analyzed by the AI with a marketing-aware description, so that the system can intelligently match my images to the right sections of my website.

## Acceptance Criteria
- [ ] Given an uploaded image, when upload completes, then AI analysis begins automatically (no user action needed)
- [ ] Given an image being analyzed, when the analysis is in progress, then the UI shows an "analyzing" spinner on the image thumbnail
- [ ] Given a completed analysis, when it renders, then the image shows an editable description, semantic tags, and dominant colors
- [ ] Given the AI description, when shown, then it is labeled "AI-generated description" with an edit icon
- [ ] Given 5 uploaded images, when all are analyzed, then analyses run in parallel via `Promise.allSettled()` completing in ~5-10 seconds total
- [ ] Given the business context (idea, industry, audience), when the image is analyzed, then the description reflects the business context (e.g., "A barista pouring latte art" for a café business, not just "person with cup")
- [ ] Given the user, when they edit an AI-generated description, then the edited text is saved and used for matching instead of the original
- [ ] Given an analysis failure for one image, when other images succeed, then failed images show a retry button and don't block the rest

## Dependencies
- US-081 (Image Upload Step)

## Technical Notes
- New service: `image-description-service.ts` using Claude Vision (multimodal LLM)
- New API endpoint: `POST /api/ai/analyze-image` accepts `{ imagePath, siteId }`
- Returns: `{ description, tags[], dominant_colors[], subject, orientation }`
- Prompt includes business context from `OnboardingSession.data` for marketing-aware descriptions
- Subject categories: person, product, place, food, abstract, group
- ADR-1 in architecture doc chose Claude Vision over dedicated Vision APIs for richer semantic descriptions
- See `architecture-user-images.md` Section 4.2 for full service specification
