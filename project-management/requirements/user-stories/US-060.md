# US-060: Content Quality Validation & Comparison Artifacts

**Epic:** Quality Assurance
**Priority:** P1
**Milestone:** M10 — Quality Framework
**PRD Refs:** FR-505, FR-506

## User Story
As a QA engineer, I want automated content uniqueness checks and before/after comparison artifacts, so that I can prove the v2 pipeline produces measurably better and more unique content.

## Acceptance Criteria
- [ ] Given content uniqueness validation, when two sites are compared, then no two sites share more than 20% of their body content verbatim (excluding common industry terminology)
- [ ] Given uniqueness measurement, when applied, then it uses Jaccard similarity on sentence-level n-grams
- [ ] Given each synthetic test case, when the v2 pipeline is run, then it generates and stores: (a) current v1 pipeline output (baseline), (b) v2 pipeline output, (c) a structured comparison showing improvements per evaluation criterion
- [ ] Given comparison artifacts, when generated, then they are stored as sprint output documentation

## Dependencies
- US-059 (Content Evaluation Rubric)
- US-057 (Synthetic Test Case Suite)

## Technical Notes
- Uniqueness validation: tokenize content into sentences, compute n-gram sets, measure Jaccard similarity
- Before/after artifacts: run both v1 and v2 pipelines on the same test case inputs
- Store comparison artifacts in `project-management/sprint-outputs/` as markdown with metrics tables
- Consider automating this as a CI job that runs on pipeline changes
