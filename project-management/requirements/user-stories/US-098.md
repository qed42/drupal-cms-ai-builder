# US-098: Design Approach Selection in Onboarding

**Epic:** Code Component Generation
**Priority:** P0
**Milestone:** M26 — Code Component Generation

## User Story
As a user completing the onboarding journey, I want to choose between a "Polished & Consistent" design system approach and a "Unique & Creative" custom-generated approach, so that the AI tailors its output to my design expectations.

## Acceptance Criteria
- [ ] Given the Style onboarding step, when the user views the page, then two design approach options are presented: "Polished & Consistent" (design_system) and "Unique & Creative" (code_components)
- [ ] Given a user selects a design approach, when they proceed through onboarding, then the selection is persisted in `generationMode` on the `OnboardingData` and in the database
- [ ] Given a user resumes onboarding, when the Style step loads, then their previous design approach selection is restored
- [ ] Given no prior selection exists, when the Style step loads, then the default is "Polished & Consistent" (design_system)
- [ ] Given the user selects "Unique & Creative", when the UI updates, then additional design preferences are optionally available (animation level, visual style, interactivity)
- [ ] Given the existing SDC pipeline, when no mode is selected or "design_system" is selected, then the existing generation flow is unchanged (backward compatible)

## Dependencies
- None
