# US-046: Markdown Content Preview Page

**Epic:** Content Review & Editing
**Priority:** P0
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-301

## User Story
As a business owner, I want to see my generated content rendered as formatted, readable text before provisioning, so that I can review what will appear on my website.

## Acceptance Criteria
- [ ] Given generation completes, when the user navigates to the review page, then all generated pages are rendered as formatted markdown (not raw JSON)
- [ ] Given the review page, when it loads, then each page is displayed as a collapsible section with: page title, meta description, section headings with component type labels (e.g., "[Hero Banner]", "[Text Block]", "[Card Grid]"), full content text (not truncated), and form field previews
- [ ] Given the review page, when all content is rendered, then it is read-only by default
- [ ] Given the review page, when it loads, then it renders within 2 seconds (NFR-05)
- [ ] Given multiple pages, when the user navigates them, then a collapsible navigation sidebar shows all pages

## Dependencies
- US-041 (Per-Page Content Generation)

## Technical Notes
- New route: `/onboarding/review` inserted between generation and provisioning
- Blueprint JSON must be transformed into renderable markdown — create a `blueprintToMarkdown()` utility
- Component type labels help users understand the site structure without seeing the actual components
- Consider using a markdown rendering library (e.g., `react-markdown`) for consistent formatting
