# US-027: Form Submission Storage & Notifications

**Epic:** Lead Capture & Forms
**Priority:** P0
**Milestone:** M4 — Lead Capture & Forms

## User Story
As a business owner, I want form submissions stored in my Drupal site and to receive email notifications, so that I never miss a lead.

## Acceptance Criteria
- [ ] Given a visitor submits a contact form, when the submission is saved, then all form data is stored in Drupal (Webform submissions or custom entity)
- [ ] Given a new form submission, when saved, then the site owner receives an email notification with the submission details
- [ ] Given multiple form submissions, when the owner logs into Drupal, then they can view all submissions in a list/table
- [ ] Given a submission, when viewed in the dashboard, then the owner can see all submitted field values and the submission timestamp
- [ ] Given email notifications, when configured, then the notification includes: submitter name, email, message preview, and a link to view the full submission

## Dependencies
- US-026 (Contact Form Auto-Generation)

## Technical Notes
- FR-403, FR-404, FR-405
- Webform module handles submission storage and email notifications natively
- Email configuration via Drupal Mail API — ensure SMTP/transactional email service is configured
- Submission view built as a Drupal View or Webform's built-in results page
