# US-075: Unique Stock Images Per Section — No Duplicates on a Page

**Epic:** Content Quality & SEO
**Priority:** P1
**Milestone:** M21 — Content Quality Hardening

## User Story
As a site owner, I want every section on my generated website to have a unique, contextually relevant stock image, so that the site looks professional and doesn't appear auto-generated.

## Acceptance Criteria
- [ ] Given a page with multiple image-bearing sections, when images are resolved from Pexels, then no two sections on the same page receive the same photo
- [ ] Given the image resolver processes a section, when a Pexels query returns a photo already used on the same page, then the resolver fetches the next available photo instead
- [ ] Given similar sections (e.g., two team cards) produce identical search queries, when images are resolved, then each section still receives a distinct image
- [ ] Given the Pexels API, when images are fetched, then there is no artificial cap on the number of images per page — every image-capable component gets a unique image
- [ ] Given a Pexels query returns only one result and it's already used, when a fallback is needed, then the resolver modifies the query (appends section context) and retries before giving up
- [ ] Given the enhance phase runs post-generation, when it processes image intents, then it also respects the per-page deduplication constraint

## Dependencies
- None — fixes existing pipeline behavior

## Technical Notes
- `stock-image-service.ts` uses `per_page=1` — identical queries always return the same photo
- In-memory cache keyed by query string accelerates duplicates rather than preventing them
- Fix: track used photo IDs per page in the resolver; request `per_page=3-5` and pick the first unused one
- `image-intent.ts` builds queries deterministically from props — similar sections produce identical queries
- Consider adding section-type hints to search queries for natural diversity (e.g., "dental office reception" vs "dental office treatment")
