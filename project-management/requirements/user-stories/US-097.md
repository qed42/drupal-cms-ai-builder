# US-097: Review Page Messaging & Layout Fixes

**Epic:** UX Polish
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a user who just finished the build process, I want the review page to set accurate expectations and have a clean action area, so that I'm not confused by misleading messaging or awkward layouts.

## Acceptance Criteria
- [ ] After clicking "Review Your Website", the progress page no longer shows "Your site is ready" — replace with contextual messaging like "Review your content before launching"
- [ ] The review page heading reflects that the user is reviewing a draft, not a finished site
- [ ] The action buttons section (top-right of review page) is redesigned to not look awkward — consult `/designer` for the right layout
- [ ] Action buttons have clear visual hierarchy (primary action prominent, secondary actions subdued)

## Dependencies
- None

## Technical Notes
- Progress page: the `getHeading()` function returns "is ready!" when `done` is true — change to a review-oriented message
- Review page: check `app/onboarding/review/page.tsx` for the action button layout
- The "ready" message is appropriate only after provisioning completes (site is actually live)
