# US-090: Split-Pane Layout for AI-Heavy Onboarding Steps

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency
**Requirement:** architecture-split-pane-onboarding.md

## User Story
As a user completing the onboarding journey, I want Archie's AI analysis displayed side-by-side with my input on desktop, so that I can see how the AI interprets my answers without scrolling past my work area.

## Acceptance Criteria
- [ ] Given a desktop viewport (≥1024px), when the user is on an AI-heavy step (idea, audience, pages, brand, fonts, follow-up, tone), then the layout renders as a 50/50 two-column grid with input on the left and Archie's analysis on the right
- [ ] Given a desktop viewport, when the user is on a simple step (start, theme, name, design, images), then the layout remains centered single-column
- [ ] Given the right pane, when AI analysis has not yet loaded, then an empty state displays with Archie's avatar and placeholder text ("I'll share my thoughts as you type...")
- [ ] Given the right pane, when AI analysis is loading, then a skeleton loader displays vertically centered in the pane
- [ ] Given the right pane, when AI analysis is populated, then the InferenceCard renders with sticky positioning (`sticky top-12`) so it remains visible during left-pane scroll
- [ ] Given the user taps "Looks right" on an InferenceCard in split mode, when the card is confirmed, then it collapses to a compact single-line summary (e.g., "✓ Industry: Dental · HIPAA · 3 services") rather than disappearing
- [ ] Given a mobile viewport (<1024px), when any step renders, then the layout falls back to single-column with InferenceCard stacked below the input (current behavior preserved)
- [ ] Given the right pane container, when it transitions from empty to populated, then content animates in with `opacity` + `translateX(8px)` over 300ms, respecting `prefers-reduced-motion`
- [ ] Given the review-settings step, when it renders, then it continues to use the existing "summary" layout mode unchanged
