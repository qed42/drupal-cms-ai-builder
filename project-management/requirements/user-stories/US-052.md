# US-052: Approve & Provision Flow

**Epic:** Content Review & Editing
**Priority:** P0
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-307

## User Story
As a business owner, I want to explicitly approve my reviewed content before provisioning begins, so that I have confidence the site will be built with content I've verified.

## Acceptance Criteria
- [ ] Given the review page, when the user has reviewed content, then a prominent "Approve & Build Site" button is visible
- [ ] Given the approve button, when the user has not viewed all pages, then the button is disabled with a hint to review remaining pages
- [ ] Given page viewing, when tracked, then viewing is detected via scroll tracking or explicit page-by-page acknowledgment
- [ ] Given the user clicks "Approve & Build Site", when clicked, then provisioning begins using the reviewed/edited blueprint
- [ ] Given the approve flow, when provisioning starts, then the user is redirected to the provisioning progress page

## Dependencies
- US-046 (Markdown Content Preview Page)
- US-047 (Inline Content Editing)

## Technical Notes
- The "viewed all pages" requirement prevents users from missing content issues
- Consider a simpler approach: checkboxes per page ("I've reviewed this page") rather than scroll tracking
- Open Question OQ-4: Should there be a "Skip Review" option? Recommend including it with a warning: "Your content hasn't been reviewed. You can edit it later in Drupal Canvas."
- The approve action should update the site status from "review" to "provisioning"
