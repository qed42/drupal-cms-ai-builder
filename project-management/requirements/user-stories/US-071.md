# US-071: Input Impact Summary on Dashboard Site Card

**Epic:** Transparent AI Onboarding
**Priority:** P3
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 5)

## User Story
As a returning user viewing my dashboard, I want to see a compact summary of how the AI used my inputs to shape the site, so that I'm reminded the AI listened to me and feel confident in the result.

## Acceptance Criteria
- [ ] Given a user views their site card on the dashboard, when the card renders, then it includes a "Your inputs shaped" section with 3-5 bullet points
- [ ] Given the bullet points, when they are displayed, then each traces a specific AI decision to a user input or industry inference (e.g., "Warm tone across all pages", "HIPAA compliance notices added", "FAQ page added — common for dental practices")
- [ ] Given the bullet points, when they are generated, then at least one references something the AI added proactively (e.g., compliance, FAQ, industry-standard page) to demonstrate AI value beyond user instructions
- [ ] Given the summary, when it is displayed, then it is visible without expanding or clicking — it's part of the default card view
- [ ] Given a site is regenerated, when the new blueprint completes, then the summary bullets update to reflect the new generation
- [ ] Given the summary bullets, when they are stored, then they are generated during the pipeline and persisted with the Blueprint record (no runtime AI call on dashboard load)

## Dependencies
- US-070 (Store Pipeline Metadata) — needs ResearchBrief and ContentPlan data to generate bullets

## Technical Notes
- Generate 3-5 bullets at the end of the pipeline using template logic:
  - Tone bullet: always include, derived from ResearchBrief.toneGuidance.primary
  - Compliance bullet: include if compliance_flags are non-empty
  - Proactive page bullet: include if AI added a page the user didn't explicitly request
  - SEO bullet: "X SEO keywords targeted across Y pages"
  - Industry bullet: "Content tailored for [industry]"
- Store as `blueprint.payload._impactSummary: string[]`
- Dashboard API already returns blueprint status — add `impactSummary` to the response.
