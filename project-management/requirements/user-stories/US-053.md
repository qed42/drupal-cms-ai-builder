# US-053: Blueprint & Content Download

**Epic:** Content Review & Editing
**Priority:** P1
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-308

## User Story
As a business owner, I want to download my generated content in various formats, so that I can use it outside the platform or share it for review.

## Acceptance Criteria
- [ ] Given the review page, when the user clicks a download option, then they can download the blueprint as a JSON file (existing feature preserved)
- [ ] Given the review page, when the user clicks "Download as Markdown", then all page content is downloaded as markdown files in a ZIP archive
- [ ] Given the review page, when the user clicks "Download Summary", then the research brief and content plan are downloaded as a PDF summary

## Dependencies
- US-046 (Markdown Content Preview Page)

## Technical Notes
- JSON download is an existing feature — ensure it continues to work
- ZIP archive: use a client-side library (e.g., `jszip`) to bundle markdown files
- PDF generation: use a server-side library (e.g., `puppeteer` or `@react-pdf/renderer`) or keep it simple with a styled HTML-to-PDF approach
- Consider deferring PDF to a later sprint if it's complex — JSON and markdown downloads provide the core value
