# US-016: Brand Token Application

**Epic:** AI Website Generation
**Priority:** P0
**Milestone:** M2 — AI Site Generation Engine

## User Story
As a business owner, I want my brand colors, fonts, and logo applied across the entire generated site, so that the website feels like mine — not a generic template.

## Acceptance Criteria
- [ ] Given the user selected brand colors, when the site is generated, then primary, secondary, and accent colors are applied as CSS custom properties
- [ ] Given the user selected fonts, when the site is generated, then the heading and body fonts are applied across all pages
- [ ] Given the user uploaded a logo, when the site is generated, then the logo appears in the site header/navigation
- [ ] Given the user provided no brand input, when the site is generated, then industry-appropriate defaults are applied
- [ ] Given brand tokens are applied, when viewed on mobile and desktop, then branding is consistent across breakpoints

## Dependencies
- US-007 (Brand Input)
- US-013 (Page Generation)

## Technical Notes
- FR-206
- Space theme's design tokens (CSS custom properties) are the mechanism for brand application
- Generate a theme settings override or sub-theme configuration with user's brand values
- Font loading via Google Fonts or self-hosted — consider performance impact (CWV)
