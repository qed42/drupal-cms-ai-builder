# US-074: Marketing-Grade Hero Headlines Instead of Generic "Welcome" Text

**Epic:** Content Quality & SEO
**Priority:** P0
**Milestone:** M21 — Content Quality Hardening

## User Story
As a site owner, I want the hero section on every page to display a compelling, marketing-grade headline derived from my business details, so that visitors immediately understand my value proposition and the page ranks well for target keywords.

## Acceptance Criteria
- [ ] Given a generated site, when the hero billboard renders, then the h1 heading NEVER contains "Welcome", "Welcome to [Name]", "Home", "About Us", or any generic greeting
- [ ] Given the tree-builder encounters a hero-billboard without a heading child, when it creates a fallback heading, then it uses the business name + tagline (not "Welcome")
- [ ] Given the AI generates a heading child in hero_slot, when the tree-builder processes it, then the child's heading_text is preserved and the fallback path is NOT triggered
- [ ] Given the content review agent validates a page, when the hero heading contains a forbidden phrase, then it flags an error-severity issue that triggers retry
- [ ] Given a content plan produced by the plan phase, when a hero section heading contains "Welcome" or generic text, then the plan is rejected and regenerated
- [ ] Given any generated page, when all text props are examined, then every text value references the business's industry, services, audience, or differentiators — not generic filler

## Dependencies
- None — fixes existing pipeline behavior

## Technical Notes
- Root cause is a hardcoded `|| "Welcome"` fallback in `packages/ds-mercury/src/tree-builders.ts:874`
- Secondary cause: `review.ts` checks `hero.props.title` and `hero.props.heading` but Mercury uses `heading_text` — review can't catch the fallback
- Prompt-level fixes already applied in plan.ts, page-generation.ts, and prompt-fragments.ts — this story covers the code-level fixes
