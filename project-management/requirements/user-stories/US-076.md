# US-076: Automated Design Review Pass in Pipeline

**Epic:** Content Quality & SEO
**Priority:** P1
**Milestone:** M21 — Content Quality Hardening

## User Story
As a site owner, I want the AI pipeline to automatically verify visual aesthetics (background colors, image placement, component variety, and heading quality), so that every generated page looks professionally designed without manual review.

## Acceptance Criteria
- [ ] Given a generated page, when the design review runs, then it flags consecutive sections with the same background color as a warning
- [ ] Given a generated page, when text+image sections appear near each other, then the review verifies image position alternates (left/right)
- [ ] Given a generated page, when consecutive sections use the same composition pattern, then the review flags it as a warning
- [ ] Given the hero section, when its heading_text matches a forbidden phrase list (Welcome, Welcome to *, Home, About Us, Our Services), then the review flags an error-severity issue
- [ ] Given the review agent, when design checks produce error-level failures, then the page is retried (same retry mechanism as content depth checks)
- [ ] Given the review agent, when design checks produce only warnings, then the page is accepted but warnings are logged for transparency
- [ ] Given the review completes, when results are reported, then design check results are included in the existing quality score

## Dependencies
- US-074 (hero headline fix provides the forbidden-phrase list to check against)

## Technical Notes
- ADR-010 mandates the review agent stays deterministic (no extra AI calls). All checks here are rule-based.
- Extend existing `review.ts` with a new `designChecks` dimension alongside contentDepth, seo, and geo
- Background color checks: walk section array, compare consecutive `container_background` or `background_color` props
- Image alternation: check `pattern` field for text-image splits, verify alternating order
- Component variety: check consecutive `component_id` and `pattern` fields for duplicates
- Keep checks fast (<10ms) — pure array traversal, no I/O
