# US-111: Live Preview Frame for Code Components

**Epic:** Live Blueprint Preview
**Priority:** P0
**Milestone:** M28 — Live Blueprint Preview

## User Story
As a user reviewing my generated website, I want to see the code component sections rendered as live, interactive React components in a visual preview, so that I can assess the actual design quality before approving and provisioning to Drupal.

## Acceptance Criteria
- [ ] Given a blueprint with code_components generation mode, when the review page loads, then each section's JSX is transpiled and rendered as a live React component inside a sandboxed iframe
- [ ] Given a code component with brand tokens (colors, fonts), when rendered in the preview, then CSS custom properties are applied and the component uses the correct brand colors and fonts
- [ ] Given a code component that references Tailwind CSS classes, when rendered in the preview, then all Tailwind styles are correctly applied via the Tailwind CDN
- [ ] Given a code component JSX that contains a runtime error, when rendered in the preview, then an error boundary catches the error and displays a styled placeholder with a "Regenerate" button — without crashing the rest of the page preview
- [ ] Given the preview is loaded, when the user scrolls, then sections below the fold are lazily rendered via IntersectionObserver for performance
- [ ] Given a page with 8 sections, when the preview renders, then the total time to first meaningful paint is under 1 second

## Dependencies
- M26 Code Component Generation (code components must exist in the blueprint)
- `sucrase` npm package (new dependency)

## Technical Notes
- Architecture: `architecture-live-blueprint-preview.md` Sections 3.1, 3.2, 3.4
- New file: `platform-app/src/lib/preview/build-preview-html.ts` — constructs iframe srcdoc
- New file: `platform-app/src/lib/preview/transpile.ts` — Sucrase JSX transpilation with caching
- New file: `platform-app/src/app/onboarding/review/components/LivePreviewFrame.tsx` — iframe wrapper
- The iframe uses `sandbox="allow-scripts"` without `allow-same-origin` for security isolation
- React and Sucrase are loaded as UMD scripts inside the iframe
- Component props are injected from `section.props` and rendered via `React.createElement`
- Each section is wrapped in an error boundary and a `data-section-index` wrapper for interaction
