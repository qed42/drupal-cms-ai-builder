# US-064: AI Inference Cards on Input Steps

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 1)

## User Story
As a user filling in the onboarding wizard, I want to see what the AI inferred from my input immediately after I provide it, so that I can verify the AI understood me correctly before moving on.

## Acceptance Criteria
- [ ] Given a user has entered a valid business idea (Idea step), when the field validates as "good", then an inference card appears within 2 seconds showing: detected industry, key services/keywords, and compliance flags (if any)
- [ ] Given a user has entered their target audience (Audience step), when the field validates, then an inference card appears showing: primary audience label, 2-3 inferred pain points, and demographic notes
- [ ] Given a user has selected a tone (Tone step), when the selection is made, then an inference card appears showing: tone characteristics and 1-2 example sentences in that tone
- [ ] Given an inference card is displayed, when the user reads it, then a plain-language sentence explains which downstream decisions this input affects (e.g., "This shapes your page suggestions and SEO keywords")
- [ ] Given an inference card is displayed, when the user clicks "Looks right", then the card dismisses and the user can proceed to next step
- [ ] Given an inference card is displayed, when the user clicks "Edit my description", then focus scrolls back to the input field
- [ ] Given any inference card, when it is displayed, then it uses NO additional API calls — it surfaces data already returned by existing validation/analyze endpoints
- [ ] Given the user is on mobile, when an inference card appears, then it renders as a dismissible bottom sheet
- [ ] Given the user is on steps without AI inference (Name, Design, Brand, Fonts), then no inference card appears

## Dependencies
- None — uses existing API response data from `/api/ai/analyze` and `/api/ai/validate-idea`

## Technical Notes
- The `/api/ai/analyze` endpoint already returns industry, keywords, and compliance_flags — currently only used to populate follow-up questions. Surface this data as an inference card on the Idea step.
- Audience pain points are inferred during the Research phase, NOT during input. Consider adding a lightweight inference to the `/api/ai/suggest-audiences` response, or derive pain points client-side from industry + audience text.
- Tone examples exist in `tone-samples.ts` — can be rendered directly without an API call.
- Card component should be reusable: `<InferenceCard title items explanation onConfirm onEdit />`
