# US-072: Inference Card UI Component

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 1, component)

## User Story
As the onboarding steps, I need a reusable InferenceCard component that displays AI insights in a consistent, non-blocking format, so that inference cards render identically across all steps that use them.

## Acceptance Criteria
- [ ] Given the InferenceCard component, when rendered, then it displays: a title (e.g., "AI understood"), a list of labeled items (key-value or bullet format), an explanation sentence, and two action buttons ("Looks right" / "Edit")
- [ ] Given the component is on desktop, when rendered, then it appears as a card below the input field or in the right panel of a split layout
- [ ] Given the component is on mobile, when rendered, then it appears as a bottom sheet with a drag handle, dismissible by swipe-down or "Looks right" tap
- [ ] Given the component receives an `isLoading` prop, when true, then it shows a skeleton with pulsing placeholder lines
- [ ] Given the component receives items, when each item has a `label` and `value`, then they render as a clean key-value list (e.g., "Industry: Family Dental Practice")
- [ ] Given the component receives items, when an item has a `list` type, then it renders as a bulleted sub-list (e.g., "Pain points: finding a good dentist, cost concerns, scheduling")
- [ ] Given the component, when it mounts with an animation, then it slides in from the bottom (mobile) or fades in from below (desktop) with a 200ms ease transition
- [ ] Given the component, when the user has not interacted with it for 30 seconds, then it auto-dismisses with a fade-out (does not block progression)

## Dependencies
- None — pure UI component

## Technical Notes
- Props interface:
  ```typescript
  interface InferenceCardProps {
    title: string;
    items: Array<{ label: string; value: string | string[]; type?: 'text' | 'list' }>;
    explanation: string;
    onConfirm: () => void;
    onEdit: () => void;
    isLoading?: boolean;
    editLabel?: string; // default: "Edit my description"
  }
  ```
- Use Headless UI or Radix for the mobile bottom sheet behavior.
- Ensure the card doesn't shift layout — use CSS `position: relative` within the step layout, not absolute/fixed on desktop.
