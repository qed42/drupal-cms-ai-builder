# US-106: Design Rules Engine — Cascading Industry & Persona Rules

**Epic:** Design Quality & Consistency
**Priority:** P1
**Milestone:** M27 — Design Rules Engine

## User Story
As a site builder, I want the AI to automatically apply industry-specific and persona-appropriate design rules, so that generated sites follow best practices for my domain without manual configuration.

## Acceptance Criteria
- [x] Given a healthcare industry selection, when code components are generated, then empathetic tone, soft CTAs, HIPAA disclosures, and calming visual direction are applied
- [x] Given a portfolio/creative industry, when code components are generated, then editorial layouts, dramatic typography, and gallery sections are prioritized
- [x] Given an enterprise persona, when code components are generated, then max sections increase to 10 and professional terminology is preferred
- [x] Given an unknown industry, when code components are generated, then global baseline rules still apply (hero + CTA required, WCAG AA)
- [x] Given the feature flag `ENABLE_DESIGN_RULES` is false, when code components are generated, then no design rules are injected (backward compatible)
- [x] Unit tests cover persona inference, cascade resolution, and prompt compilation (28 tests)

## Dependencies
- US-101 (Designer Agent)
- US-102 (Pipeline Mode Branching)

## Technical Notes
- Rules stored as YAML in `platform-app/src/lib/rules/definitions/`
- Cascade order: global → industry → persona
- Arrays concatenate and deduplicate; scalars overwrite
- Feature-flagged via `ENABLE_DESIGN_RULES=true` environment variable
