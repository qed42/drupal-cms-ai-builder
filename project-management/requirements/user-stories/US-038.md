# US-038: AI Provider Abstraction Layer

**Epic:** AI Content Pipeline
**Priority:** P0
**Milestone:** M7 — AI Content Pipeline
**PRD Refs:** FR-205, FR-206

## User Story
As a platform developer, I want a model-agnostic AI provider abstraction, so that I can switch between OpenAI and Anthropic (or use different models per phase) without code changes.

## Acceptance Criteria
- [ ] Given environment variables `AI_PROVIDER` and `AI_MODEL`, when the platform starts, then it initializes the correct AI provider (openai or anthropic)
- [ ] Given the abstraction layer, when any component calls `generateJSON(prompt, schema)`, then it returns a typed object regardless of the underlying provider
- [ ] Given the abstraction layer, when any component calls `generateText(prompt)`, then it returns a string regardless of the underlying provider
- [ ] Given `AI_PROVIDER=anthropic` and `AI_MODEL=claude-sonnet-4-20250514`, when a generation is triggered, then it uses the Anthropic API with the specified model
- [ ] Given optional per-phase env vars (`AI_MODEL_RESEARCH`, `AI_MODEL_PLAN`, `AI_MODEL_GENERATE`), when set, then each pipeline phase uses its configured model override
- [ ] Given the abstraction layer, when the provider returns a rate limit (HTTP 429), then it implements exponential backoff with jitter (max 3 retries)
- [ ] Given a provider switch, when only environment variables are changed (and container restarted), then the system works without any code changes

## Dependencies
- None (foundation story for the pipeline)

## Technical Notes
- Replaces the current hardcoded OpenAI `gpt-4o-mini` usage in `platform-app/src/lib/blueprint/generator.ts`
- Interface: `AIProvider { generateJSON<T>(prompt, schema, options?): Promise<T>; generateText(prompt, options?): Promise<string> }`
- `GenerateOptions`: `{ model?, temperature?, maxTokens?, retries? }`
- Both providers support structured output (OpenAI via function calling, Anthropic via tool use)
- Cost tracking per call would be valuable for monitoring the $0.50/site budget constraint
