# US-105: Code Component Preview in Review Editor

**Epic:** Code Component Generation
**Priority:** P1
**Milestone:** M26 — Code Component Generation

## User Story
As a user reviewing my generated site before publishing, I want to see a meaningful preview of code component sections in the review editor, so that I can assess the design quality and request regeneration if needed.

## Acceptance Criteria
- [ ] Given a generated site with code components, when the review editor loads, then code component sections display a rendered preview (not raw code)
- [ ] Given a code component section in the review editor, when the user clicks on it, then they can see the JSX source with syntax highlighting for inspection
- [ ] Given a code component section in the review editor, when the user clicks "Regenerate", then the Designer Agent generates a new version of that section with the same brief
- [ ] Given a mixed-mode blueprint (SDC + Code Components), when the review page renders, then both section types display correctly in their respective preview formats
- [ ] Given a code component with validation warnings (e.g., accessibility), when displayed in the review editor, then warnings are surfaced as non-blocking indicators

## Dependencies
- US-101 (Designer Agent for regeneration)
- US-102 (pipeline produces code component sections)
