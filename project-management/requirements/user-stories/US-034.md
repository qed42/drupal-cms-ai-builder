# US-034: Key Differentiators & Tone Selection

**Epic:** Onboarding Enhancements
**Priority:** P0
**Milestone:** M6 — Onboarding Enrichment
**PRD Refs:** FR-102, FR-104

## User Story
As a business owner, I want to describe what makes my business unique and select my preferred writing tone, so that the generated content reflects my brand voice and competitive advantages.

## Acceptance Criteria
- [ ] Given the onboarding wizard, when the user reaches the differentiators step, then they see a text input asking "What makes your business different from competitors?" with a placeholder example relevant to their detected industry
- [ ] Given the differentiators input, when the user types their response, then it is passed to the Research phase as a primary signal
- [ ] Given the tone selection step, when the user reaches it, then they see 3-4 writing samples (2-3 sentences each) representing: Professional/Formal, Warm/Friendly, Bold/Confident, Casual/Conversational
- [ ] Given tone samples, when the user selects one, then the selected tone replaces the current auto-detected `tone` field in the blueprint generation
- [ ] Given both inputs, when onboarding completes, then differentiators and tone selection are stored alongside other onboarding data

## Dependencies
- US-006 (Industry Selection)
- US-008 (Business Context)

## Technical Notes
- Tone samples should be industry-aware if possible (e.g., healthcare samples use medical language)
- The differentiators text is a primary input to the Research phase prompt — it directly shapes content uniqueness
- Consider combining differentiators and tone into a single onboarding step to minimize wizard length
