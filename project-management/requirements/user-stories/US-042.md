# US-042: Pipeline Phase Visibility UI

**Epic:** AI Content Pipeline
**Priority:** P0
**Milestone:** M7 — AI Content Pipeline
**PRD Refs:** FR-204

## User Story
As a business owner, I want to see each generation phase (Research, Plan, Generate) progressing in real time, so that I understand what the AI is doing and trust the process.

## Acceptance Criteria
- [ ] Given generation starts, when the user is on the progress page, then they see three phases: Research, Plan, Generate displayed as a step-by-step progress view
- [ ] Given each phase, when it is running, then it shows a status indicator: pending, in progress, complete, or failed
- [ ] Given each phase, when it is in progress, then it shows estimated and elapsed time
- [ ] Given each phase, when it completes, then it shows a summary of what was produced (expandable)
- [ ] Given the Research phase completes, when the summary expands, then the user sees key findings from the research brief
- [ ] Given the Plan phase completes, when the summary expands, then the user sees the content plan outline
- [ ] Given the Generate phase, when individual pages are being generated, then per-page progress is shown (e.g., "Generating Home page... Generating Services page...")
- [ ] Given the new progress UI, when it renders, then it replaces the current single-spinner progress page

## Dependencies
- US-039 (Research Phase)
- US-040 (Plan Phase)
- US-041 (Per-Page Content Generation)

## Technical Notes
- This replaces the current `onboarding/progress/page.tsx` single-spinner
- Requires API changes to return phase-level status, not just site-level status
- Consider SSE (Server-Sent Events) or polling with a `currentPhase` + `phaseStatus` field
- Each phase summary should be collapsible to avoid overwhelming the user
