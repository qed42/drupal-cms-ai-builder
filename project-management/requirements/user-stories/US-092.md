# US-092: Adopt shadcn/ui Primitives for Platform App

**Priority:** P1
**Milestone:** M24 — UI Component System

## User Story

As a **user navigating the onboarding wizard and dashboard**, I want **consistent, accessible UI controls** so that **I can interact with all elements via keyboard, receive visible error feedback, and experience a polished interface**.

## Background

The platform-app has 23 custom components using raw Tailwind with no shared primitives. This has led to inconsistent button sizing, missing keyboard navigation on menus/selects, no focus traps on overlays, and errors that only appear in the browser console.

## Acceptance Criteria

- [ ] shadcn/ui initialized with project's dark theme and brand/accent color tokens
- [ ] Button, Input, Textarea, Select, Card, Badge, Skeleton components replace ad-hoc implementations
- [ ] Toast/Sonner provides user-facing error and success notifications
- [ ] All buttons and inputs have consistent sizing variants (sm/default/lg)
- [ ] FontSelector uses shadcn Select with keyboard navigation
- [ ] SiteCard menu uses DropdownMenu with focus trap and escape handling
- [ ] No visual regression in existing onboarding flow
- [ ] Bundle size increase < 30KB gzipped
