# US-035: Reference URL & Existing Copy Input

**Epic:** Onboarding Enhancements
**Priority:** P1
**Milestone:** M6 — Onboarding Enrichment
**PRD Refs:** FR-103, FR-105

## User Story
As a business owner, I want to optionally provide reference website URLs and paste existing marketing copy, so that the AI can match the style and incorporate my existing messaging.

## Acceptance Criteria
- [ ] Given the onboarding wizard, when the user reaches the reference input step, then they see an optional field for 1-3 URLs of websites they admire
- [ ] Given reference URLs, when provided, then they are stored but never scraped without user consent
- [ ] Given reference URLs, when passed to the Research phase, then they are included as text context for tone and structure analysis
- [ ] Given the existing copy input, when the user reaches it, then they see an optional textarea (max 2,000 characters) for pasting existing marketing copy, taglines, or About text
- [ ] Given existing copy, when provided, then it is passed verbatim to the generation phase as reference material the AI should incorporate or build upon
- [ ] Given both fields are optional, when the user skips them, then the pipeline proceeds without these inputs

## Dependencies
- US-034 (Key Differentiators & Tone Selection)

## Technical Notes
- URLs are stored as-is — no web scraping in v2 (see PRD Out of Scope)
- Existing copy paste should be sanitized (strip HTML, limit length) before storage
- These are enrichment inputs — the pipeline must work without them
