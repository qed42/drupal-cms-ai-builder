# US-093: Engaging AI Progress Experience ("Archie's Workshop")

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a site builder, I want to see Archie's reasoning and live content previews while my site is being built, so that the 2-3 minute wait feels engaging rather than passive.

## Acceptance Criteria
- [ ] Given the progress page, when a pipeline phase completes, then the phase card expands to show what was produced (industry detected, pages planned, content preview, images matched)
- [ ] Given the generate phase is active, when content is being written, then a live preview of the current page's opening text scrolls into view
- [ ] Given desktop viewport (>=1024px), when the progress page loads, then a right-column "Activity Log" shows Archie's reasoning messages as a scrolling feed
- [ ] Given mobile viewport (<1024px), when the progress page loads, then the last 3 reasoning messages appear inline below the phase cards
- [ ] Given any phase emits a reasoning message, when it appears in the log, then it fades in with a 200ms opacity animation (respects prefers-reduced-motion)
- [ ] Given all phases complete, when the completion state renders, then a brief celebration animation plays before the "Review" CTA appears
