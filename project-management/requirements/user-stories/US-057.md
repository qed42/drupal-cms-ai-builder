# US-057: Synthetic Test Case Suite & Manual Scripts

**Epic:** Quality Assurance
**Priority:** P0
**Milestone:** M10 — Quality Framework
**PRD Refs:** FR-501, FR-502

## User Story
As a QA engineer, I want a suite of synthetic business scenarios with step-by-step test scripts, so that I can systematically evaluate content quality across industries.

## Acceptance Criteria
- [ ] Given the test suite, when maintained, then it contains 10-15 synthetic business scenarios as structured JSON/YAML files
- [ ] Given each test case, when defined, then it includes: persona name, business type, idea input, industry, expected pages, content quality expectations, and evaluation criteria
- [ ] Given test case coverage, when reviewed, then it covers: healthcare, legal, restaurant, SaaS, wellness, real estate, food service, mental health, nonprofit, and event planning
- [ ] Given each test case, when documented, then a step-by-step manual test script exists covering: exact onboarding inputs, expected research brief outputs, expected content plan structure, content quality checkpoints (word count, terminology, CTA relevance), and pass/fail criteria
- [ ] Given manual test scripts, when stored, then they are maintained as markdown files in `project-management/test-cases/`

## Dependencies
- US-039 (Research Phase) — test cases validate Research output
- US-040 (Plan Phase) — test cases validate Plan output
- US-041 (Per-Page Content Generation) — test cases validate Generate output

## Technical Notes
- Test case JSON schema should match the onboarding data model for direct API injection
- Include both "happy path" scenarios and edge cases (very short idea text, unusual industries)
- Test scripts should be executable by someone unfamiliar with the system
- Consider test case difficulty levels: 5 "core" industries + 5 "stretch" industries
