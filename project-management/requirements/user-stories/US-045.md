# US-045: Phase Retry & Re-run

**Epic:** AI Content Pipeline
**Priority:** P1
**Milestone:** M7 — AI Content Pipeline
**PRD Refs:** FR-209

## User Story
As a business owner, I want to re-run any individual generation phase without restarting the entire pipeline, so that I can improve specific outputs without losing work from other phases.

## Acceptance Criteria
- [ ] Given the pipeline UI, when a phase has completed, then the user can click "Re-run" on that phase
- [ ] Given re-running the Research phase, when it completes, then the Plan and Generate outputs are invalidated (marked stale)
- [ ] Given re-running the Plan phase, when it completes, then only the Generate output is invalidated
- [ ] Given re-running the Generate phase, when it completes, then the Research brief and Content plan are preserved unchanged
- [ ] Given a re-run, when it completes, then the previous version is retained and the new output is stored as a new version

## Dependencies
- US-044 (Pipeline Data Persistence & Versioning)
- US-042 (Pipeline Phase Visibility UI)

## Technical Notes
- Invalidation means marking downstream outputs as "stale" — they still exist but the UI warns the user they were generated from a previous version
- Re-running Research should prompt: "This will regenerate your content plan and page content. Continue?"
- Consider allowing the user to provide additional guidance when re-running a phase
