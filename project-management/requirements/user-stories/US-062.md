# US-062: Content Quality Dashboard

**Epic:** Vision & Communication
**Priority:** P2
**Milestone:** M10 — Quality Framework
**PRD Refs:** FR-603

## User Story
As a platform admin, I want a dashboard showing aggregate content quality metrics, so that I can monitor generation quality and identify areas for improvement.

## Acceptance Criteria
- [ ] Given the dashboard, when accessed by an admin, then it shows: average word count per site, average evaluation score, generation time, model usage breakdown, and failure rates
- [ ] Given the dashboard, when metrics are displayed, then they aggregate data across all generated sites
- [ ] Given the dashboard, when accessible, then it is behind admin authentication (not public)

## Dependencies
- US-059 (Content Evaluation Rubric)
- US-044 (Pipeline Data Persistence)

## Technical Notes
- This is a "Could" priority — implement only if time permits
- Could be a simple Next.js page querying the database directly
- Alternatively, start with a CLI script that outputs metrics to console — upgrade to a UI later
- Metrics source: research_briefs, content_plans, and blueprint tables with generation metadata
