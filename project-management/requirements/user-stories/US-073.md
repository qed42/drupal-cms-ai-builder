# US-073: Enrich Analyze API Response for Inference Cards

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 1, backend)

## User Story
As the Idea and Audience inference cards, I need the existing AI analysis endpoints to return richer structured data, so that inference cards can display meaningful AI insights without additional API calls.

## Acceptance Criteria
- [ ] Given a call to `/api/ai/analyze` after the Idea step, when the response returns, then it includes: `industry` (string), `keywords` (string[]), `compliance_flags` (string[]), and a new `detectedServices` (string[], 2-5 items inferred from the idea text)
- [ ] Given a call to `/api/ai/suggest-audiences`, when the response returns, then it includes a new `painPoints` field (string[], 2-3 inferred pain points based on industry + audience)
- [ ] Given the existing analyze API, when `detectedServices` is added, then the AI prompt is updated to extract service/offering mentions from the idea text
- [ ] Given the existing suggest-audiences API, when `painPoints` is added, then the AI prompt is updated to infer customer pain points from the industry + audience combination
- [ ] Given backward compatibility, when existing consumers call these endpoints, then all current fields remain unchanged — new fields are additive only
- [ ] Given the API responses, when they are consumed by inference cards, then the data is sufficient to render all card content without client-side inference

## Dependencies
- None — extends existing API endpoints

## Technical Notes
- `/api/ai/analyze` already calls the AI with industry detection. Adding `detectedServices` is a prompt modification — ask the AI to also extract service/offering mentions.
- `/api/ai/suggest-audiences` currently returns audience suggestions as strings. Adding `painPoints` requires either expanding the prompt or making a parallel lightweight inference.
- Keep the additional prompt tokens minimal — these are called during real-time input, so latency matters. Target <500ms additional latency.
- Consider whether `painPoints` should be per-suggestion or global. Recommend global (simpler, faster).
