# US-003: Trial Expiry Notifications & Handling

**Epic:** User Account & Trial Management
**Priority:** P1
**Milestone:** M5 — Publishing & Subscription

## User Story
As a trial user, I want to receive notifications before my trial expires, so that I can decide to subscribe and avoid losing my site.

## Acceptance Criteria
- [ ] Given an active trial, when 4 days remain, then the user receives an email notification
- [ ] Given an active trial, when 1 day remains, then the user receives a final reminder email with a subscribe link
- [ ] Given an expired trial, when the user's site is live, then the site is taken offline (returns a maintenance page)
- [ ] Given an expired trial, when the site goes offline, then all content and configuration data is preserved (not deleted)
- [ ] Given an expired trial, when the user logs in, then they see a clear message to subscribe with a link to do so

## Dependencies
- US-002 (Free Trial Activation)
- US-020 (One-Click Publish)

## Technical Notes
- Cron-based job to check trial expiry dates daily
- Drupal Queue API for email dispatch
- Site offline mechanism — unpublish all content or block anonymous access via middleware
