# US-040: Plan Phase with User Approval

**Epic:** AI Content Pipeline
**Priority:** P0
**Milestone:** M7 — AI Content Pipeline
**PRD Refs:** FR-202

## User Story
As a business owner, I want to see a content plan before the AI generates my website content, so that I can review the structure and provide input before full generation.

## Acceptance Criteria
- [ ] Given the Research phase completes, when the Plan phase begins, then it uses the research brief to generate a content plan
- [ ] Given the content plan, when generated, then it includes: (a) site map with page purposes, (b) per-page content outline with sections, estimated word counts, and key messages, (c) SEO keyword targets per page (2-3 primary keywords), (d) recommended CTAs per page tied to business goals, (e) tone/voice guidelines derived from user selection + industry norms
- [ ] Given the content plan, when displayed to the user, then it appears as "Your content plan" with a collapsible outline per page
- [ ] Given the content plan display, when the user reviews it, then they can approve or request modifications before generation proceeds
- [ ] Given the Plan phase, when it runs, then it completes within 15 seconds (NFR-02)
- [ ] Given the content plan, when generated, then it is stored as a separate versioned record in the database

## Dependencies
- US-038 (AI Provider Abstraction Layer)
- US-039 (Research Phase)

## Technical Notes
- Single AI call using `generateJSON()` with the research brief as context
- ContentPlan TypeScript interface with per-page outlines
- Open Question OQ-1: Should the plan require explicit approval or auto-proceed? Default to showing the plan with an "Approve & Generate" button
- Store in a new `content_plans` table with foreign key to the research brief
- The plan's per-page outlines become the generation prompts for each page
