# US-067: Live Generation Narrative on Progress Page

**Epic:** Transparent AI Onboarding
**Priority:** P2
**Milestone:** M20 — AI Transparency
**Requirement:** REQ-transparent-ai-onboarding.md (Feature 3)

## User Story
As a user waiting for my website to generate, I want to see a human-readable narrative of what the AI is doing and deciding in real time, so that the wait feels productive and I understand the AI's reasoning.

## Acceptance Criteria
- [ ] Given the Research phase completes, when its card updates on the progress page, then a 2-3 sentence summary appears explaining what was decided (e.g., "Identified your practice as family dentistry with pediatric focus. Found 3 key differentiators from your inputs.")
- [ ] Given the Plan phase completes, when its card updates, then a summary appears referencing page count and notable decisions (e.g., "Organized 6 pages with 28 content sections. Added FAQ page based on your industry.")
- [ ] Given the Generate phase is in progress, when pages are being generated, then the active card updates per-page showing: which page is being written (e.g., "Writing Services page (3 of 6)"), the tone being applied, and target keywords
- [ ] Given any phase summary, when it is displayed, then at least one sentence connects an AI decision back to a specific user input (e.g., "Using your warm, reassuring tone" not "Applying tone settings")
- [ ] Given a phase encounters an error and retries, when the retry happens, then the card shows actionable context (e.g., "Content for About page didn't meet quality bar — regenerating with more detail")
- [ ] Given all phases complete, when the completion state renders, then a final summary appears: "Generated X pages with Y words of content, Z images, optimized for N keywords"
- [ ] Given summaries are generated, when they are created, then they come from the pipeline phases themselves — no separate post-hoc AI call
- [ ] Given summaries are displayed, when the user scrolls, then completed phase summaries persist and remain visible
- [ ] Given the generation summary, when it is produced, then it is stored in the Blueprint record for later reference

## Dependencies
- None — modifies existing pipeline phases and progress UI

## Technical Notes
- Each pipeline phase already returns a `summary` field in its status update. Currently these are generic. This story makes them specific and user-input-aware.
- The `PipelineProgress.tsx` component already renders phase descriptions — it just needs richer content.
- Generate phase needs to emit per-page progress updates via the status polling endpoint. The current status endpoint returns `generationStep` — extend to include `currentPage` and `pageCount`.
- Summary text generation: construct from structured data (ResearchBrief fields, ContentPlan page count, etc.), not from an LLM call. Template-based approach.
