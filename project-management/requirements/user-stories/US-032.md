# US-032: SSL Provisioning

**Epic:** Publishing & Deployment
**Priority:** P0
**Milestone:** M5 — Publishing & Subscription

## User Story
As a site owner, I want SSL automatically provisioned for my published site, so that visitors see a secure (HTTPS) connection without me doing anything.

## Acceptance Criteria
- [ ] Given a site is published on a platform subdomain, when it goes live, then HTTPS is enabled by default
- [ ] Given a custom domain is connected, when DNS is verified, then an SSL certificate is automatically provisioned
- [ ] Given SSL is active, when a visitor accesses the site via HTTP, then they are redirected to HTTPS
- [ ] Given SSL certificates, when they approach expiry, then they are automatically renewed

## Dependencies
- US-030 (One-Click Publish)

## Technical Notes
- FR-504
- Let's Encrypt for automatic certificate provisioning
- Platform-level SSL (wildcard cert for *.platform.com subdomains)
- Custom domain SSL requires per-domain cert provisioning — depends on hosting infrastructure
