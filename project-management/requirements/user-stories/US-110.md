# US-110: Enable Design Rules by Default & Remove Feature Flag

**Epic:** Design Quality & Consistency
**Priority:** P1
**Milestone:** M27 — Design Rules Engine

## User Story
As the product team, we want design rules to be enabled by default for all code_components generation, so that every generated site benefits from visual consistency without requiring manual feature flag configuration.

## Acceptance Criteria
- [ ] Given no `ENABLE_DESIGN_RULES` environment variable is set, when code components are generated, then design rules are resolved and injected (default ON)
- [ ] Given `ENABLE_DESIGN_RULES=false`, when code components are generated, then design rules are skipped (opt-out preserved)
- [ ] Given design_system generation mode, when pages are generated, then design rules are NOT injected (SDC mode unchanged)
- [ ] End-to-end test: generate a code_components site without the feature flag set, verify tokens appear in the prompt

## Dependencies
- US-108 (Token Injection Integration)
- US-109 (P1 Quality Rules)

## Technical Notes
- Change `getDesignRules()` default: currently returns null when flag is absent, should return rules when flag is absent
- Only disable when `ENABLE_DESIGN_RULES === "false"` (explicit opt-out)
- This should be the last task in the milestone — enable after all rules are validated
