# US-100: Code Component Validation Pipeline

**Epic:** Code Component Generation
**Priority:** P0
**Milestone:** M26 — Code Component Generation

## User Story
As a platform operator, I want all AI-generated Code Components validated for structural correctness, accessibility compliance, animation safety, and security threats, so that generated sites are safe, accessible, and performant.

## Acceptance Criteria
- [ ] Given generated JSX, when it lacks a default export, then validation fails with a clear error message identifying the issue
- [ ] Given generated JSX, when it contains images without `alt` attributes, then validation reports an accessibility warning
- [ ] Given generated JSX, when heading hierarchy skips levels (e.g., h1 → h3), then validation reports an accessibility warning
- [ ] Given generated CSS/JSX with animations, when `prefers-reduced-motion` is not handled, then validation reports a motion-safety error
- [ ] Given generated JS, when it contains dangerous patterns (`eval`, `fetch`, `document.cookie`, `localStorage`, `innerHTML`), then validation fails with a security error
- [ ] Given validation results, when errors are returned, then each error includes the rule name, human-readable message, and line number — structured for LLM retry feedback
- [ ] Given a valid component, when all checks pass, then `ValidationResult.valid` is `true` with no errors (warnings are permitted)
- [ ] Given unit tests, when all tests run, then every validation rule has at least one positive and one negative test case
