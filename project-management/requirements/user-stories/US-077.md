# US-077: Conversational Step Labels & Section Names

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a first-time user going through onboarding, I want the step headings, subtitles, and progress section names to feel like a guided conversation rather than a configuration form, so that I understand how each input shapes my final website and feel like I'm collaborating with the AI.

## Acceptance Criteria
- [ ] Given the progress stepper, when it renders section names, then it shows "Your Business", "Site Structure", "Brand & Style", "Review & Build" instead of "Vision", "Design", "Content", "Launch"
- [ ] Given any onboarding step, when its heading renders, then it uses the conversational heading from the approved label mapping (e.g., "Tell us about your business" instead of "What's the big idea?")
- [ ] Given any onboarding step, when its subtitle renders, then it explains how Archie uses the input (e.g., "Describe what you do, who you serve, and what makes you different. The more detail, the better Archie can tailor your content.")
- [ ] Given the step navigation labels in `onboarding-steps.ts`, when the sidebar/stepper renders, then it shows the updated short labels (e.g., "Your Idea" instead of "Big Idea", "Voice" instead of "Tone & Voice")
- [ ] Given the start page, when it renders, then the heading is "Let's build your website together" with Archie-branded subtitle
- [ ] Given the review-settings page, when it renders, then the heading is "Review and launch" with Archie-branded subtitle

## Dependencies
- None — string changes only

## Technical Notes
- Changes span `onboarding-steps.ts` (labels, section names) and all 12 step `page.tsx` files (StepLayout title/subtitle props)
- Full label mapping is in `architecture-onboarding-ux-polish.md` Section 3
- No component changes needed — StepLayout already accepts these props
