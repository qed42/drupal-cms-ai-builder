# US-095: Progress Page Phase State Persistence & Provisioning Engagement

**Epic:** UX Polish
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a user watching my site being built, I want completed pipeline phases to stay green during provisioning and see activity log messages during the launch phase, so that I feel continuous progress rather than a reset.

## Acceptance Criteria
- [ ] After content review, "Writing your content" and "Adding images" phase cards remain green/complete during provisioning
- [ ] During provisioning, Archie's Workshop shows provisioning-stage messages (e.g., "Installing modules...", "Configuring your theme...")
- [ ] The provisioning progress section receives equal or greater visual priority compared to the earlier pipeline phase cards
- [ ] Consult `/designer` for a revised provisioning layout that elevates the launch steps above the completed content phases

## Dependencies
- US-093 (ActivityLog + pipeline messages infrastructure)

## Technical Notes
- Phase card state: the status API already returns complete for finished phases — ensure the progress page doesn't reset pipeline state when entering provisioning
- Provisioning messages: extend the `emitMessage` pattern to provisioning steps (these run in a separate flow — check `provision/start` endpoint)
- Layout priority: the provisioning card currently renders below 4 completed phase cards — consider collapsing completed phases into a summary row during provisioning
