# US-048: Per-Section AI Regeneration

**Epic:** Content Review & Editing
**Priority:** P0
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-303

## User Story
As a business owner, I want to regenerate a specific section using AI with optional guidance, so that I can improve individual parts without affecting the rest of my content.

## Acceptance Criteria
- [ ] Given the review page, when the user clicks "Regenerate" on any section, then the AI regenerates that section only
- [ ] Given the regenerate action, when triggered, then the user can optionally provide guidance text (e.g., "Make this more formal" or "Focus on pediatric services")
- [ ] Given regeneration, when the AI generates new content, then it uses the original research brief and content plan as context for consistency
- [ ] Given regeneration, when it completes, then the new section replaces the current one in the preview
- [ ] Given regeneration, when it completes, then the previous version is preserved and accessible via an "Undo" action
- [ ] Given per-section regeneration, when triggered, then it completes within 15 seconds (NFR-06)

## Dependencies
- US-046 (Markdown Content Preview Page)
- US-038 (AI Provider Abstraction Layer)

## Technical Notes
- API endpoint: `POST /api/blueprint/{id}/regenerate-section` with `{ pageIndex, sectionIndex, guidance? }`
- The regeneration prompt includes: research brief, content plan, current page context, and user guidance
- Store the previous version in memory (client-side) for undo — no need for server-side version history per section edit
- Rate limit regeneration to prevent abuse (e.g., max 10 per blueprint per hour)
