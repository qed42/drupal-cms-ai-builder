# US-047: Inline Content Editing

**Epic:** Content Review & Editing
**Priority:** P0
**Milestone:** M8 — Content Review & Editing
**PRD Refs:** FR-302

## User Story
As a business owner, I want to edit any section's text directly in the review page, so that I can fix inaccuracies or adjust wording before my site is built.

## Acceptance Criteria
- [ ] Given the review page, when the user clicks an "Edit" button on any section, then it switches to an inline editor
- [ ] Given the inline editor, when active, then it supports plain text editing for all text fields (titles, descriptions, body content)
- [ ] Given the inline editor, when active, then it supports adding/removing list items (services, team members, testimonials)
- [ ] Given the inline editor, when active, then it supports modifying form field labels and types
- [ ] Given content changes, when the user edits text, then changes are saved to the blueprint in real time (auto-save with debounce)
- [ ] Given the inline editor, when the user finishes editing, then they can click "Done" to return to read-only view

## Dependencies
- US-046 (Markdown Content Preview Page)

## Technical Notes
- v2 uses plain textarea editing (not WYSIWYG) per PRD scope — rich editing deferred to v2.1
- Auto-save with 500ms debounce to avoid excessive API calls
- Edits modify the blueprint JSON in the database — need an update API endpoint
- Consider optimistic UI updates with background persistence
