# US-102: Dual-Mode Pipeline — SDC & Code Component Branching

**Epic:** Code Component Generation
**Priority:** P0
**Milestone:** M26 — Code Component Generation

## User Story
As a user who completed onboarding with either design approach, I want the generation pipeline to automatically use the correct path — design system components or AI-generated code components — so that my site is built according to my chosen approach without manual intervention.

## Acceptance Criteria
- [ ] Given `generationMode === "design_system"`, when the Generate phase runs, then the existing SDC adapter path executes identically to current behavior (zero regression)
- [ ] Given `generationMode === "code_components"`, when the Generate phase runs, then the Designer Agent generates JSX/Tailwind sections instead of mapping to SDC components
- [ ] Given code_components mode, when each section is generated, then progress messages are emitted ("Designing hero section...", "Creating features layout...") for real-time UI feedback
- [ ] Given code_components mode, when all sections are generated, then the blueprint payload includes `_codeComponents[]` containing config YAML data alongside the component tree
- [ ] Given code_components mode, when header and footer are generated, then they are also produced as Code Components (not SDC components)
- [ ] Given a mixed pipeline run, when Research and Plan phases execute, then they are identical regardless of mode — only the Generate phase differs
- [ ] Given a generation failure in code_components mode, when the error is non-recoverable, then a clear error message is shown and the user can retry or switch to design_system mode

## Dependencies
