# US-041: Per-Page Content Generation

**Epic:** AI Content Pipeline
**Priority:** P0
**Milestone:** M7 — AI Content Pipeline
**PRD Refs:** FR-203

## User Story
As a business owner, I want the AI to generate full-length, industry-specific content for each page individually, so that my website has 2,000-5,000+ words of meaningful, contextual content.

## Acceptance Criteria
- [ ] Given an approved content plan, when the Generate phase begins, then each page is generated in a separate AI call (not all pages in one call)
- [ ] Given per-page generation, when a service description is generated, then it contains 150-300 words with value propositions, process details, and expected outcomes
- [ ] Given per-page generation, when an about page is generated, then it contains 300-500 words covering mission, history, and differentiators
- [ ] Given per-page generation, when team bios are generated, then each bio is 100-150 words with expertise, credentials, and personality
- [ ] Given per-page generation, when testimonials are generated, then each has 3-5 sentences with specific outcomes/metrics
- [ ] Given per-page generation, when FAQ content is generated, then it produces 5-10 Q&As with 100-200 word answers each
- [ ] Given per-page generation, when CTAs are generated, then they are contextual to the page purpose (not generic "Get started")
- [ ] Given total site generation, when all pages are complete, then total content is 2,000-5,000+ words depending on page count
- [ ] Given per-page generation, when each page is generated, then it completes within 20 seconds per page and < 120 seconds total for a 6-page site (NFR-03)

## Dependencies
- US-038 (AI Provider Abstraction Layer)
- US-040 (Plan Phase)

## Technical Notes
- N AI calls (1 per page) using `generateJSON()` with the research brief + content plan as context
- Each call receives the research brief, content plan, and site-level context (tone, brand) for consistency
- Output must conform to the existing blueprint component tree structure (Space DS SDC components)
- This replaces the current single-call generation in `generator.ts`
- Consider parallel page generation (Promise.allSettled) if provider rate limits allow
