# US-063: UX Revamp — Modern Onboarding Journey & Dashboard Experience

**Epic:** Onboarding UX Modernization
**Priority:** P0
**Milestone:** M18 — UX Revamp

## User Story
As a first-time user, I want an onboarding experience that feels premium, shows me how my choices shape the website in real-time, and explains what AI is doing with my inputs, so that I trust the product and feel confident about the generated result.

## Problem Statement

The current onboarding has several UX gaps:
1. **Empty space** — Steps show a single input centered on screen with large unused areas
2. **No feedback loop** — Users provide inputs but get no visual confirmation of how AI interprets them
3. **Trust deficit** — Users can't see what's happening behind the scenes; the AI feels like a black box
4. **Dashboard duplication** — Subscription cards appear duplicated, cluttering the returning-user experience
5. **Generic color scheme** — The teal palette doesn't convey the premium, AI-forward brand identity (Sprint 23 attempted a fix but Tailwind v4 `@theme inline` didn't render custom colors — needs proper research first)
6. **No progressive reveal** — All steps feel identical; there's no sense of the website "taking shape"

## Research Requirements (Pre-Implementation)

Before any design or code work, conduct research on:

### 1. SaaS Onboarding Best Practices
- Analyze onboarding flows of top 10% SaaS apps (Notion, Linear, Vercel, Framer, Webflow, Squarespace, Wix, Canva, Figma, Stripe)
- Document patterns: progressive disclosure, live previews, contextual guidance, trust signals
- Identify which patterns apply to an AI website builder context

### 2. Color Scheme & Design System
- Research Tailwind v4 custom theming — specifically how to reliably define custom color palettes (the `@theme inline` approach failed in Sprint 23; investigate `@theme` without inline, CSS layers, or direct `:root` overrides)
- Audit top SaaS color palettes for dark-mode AI products
- Propose a palette with accessibility verification (WCAG AA contrast ratios)
- Create a working spike that proves colors render correctly in the Next.js + Tailwind v4 stack

### 3. AI Transparency Patterns
- Research how AI products communicate "what the AI is thinking" (ChatGPT, Jasper, Copy.ai, Midjourney)
- Identify lightweight patterns: streaming text summaries, confidence indicators, "AI is learning X from your input" cards

### 4. Live Preview UX
- Research split-pane and side-panel preview patterns in builders (Webflow, Framer, Squarespace)
- Evaluate feasibility of a lightweight preview panel showing how choices affect the site layout

## Acceptance Criteria

### Onboarding Journey
- [ ] Given a user is on any onboarding step, when they provide input, then they see a contextual feedback element (e.g., "AI insight" card, preview panel, or summary strip) showing what AI learned from that input
- [ ] Given a user is on the pages/design/brand steps, when selections are made, then a progressive site preview updates to reflect choices (layout skeleton, color swatches applied, font preview)
- [ ] Given a user is on any step, when they look at the page, then empty space is utilized (either split layout with preview, or contextual cards filling the side)
- [ ] Given a user completes the onboarding, when they reach the generation screen, then they see a detailed breakdown of what AI will generate (page list, tone, colors, fonts) as a "recipe card"
- [ ] Given the onboarding uses brand colors, when rendered in browser, then all custom colors display correctly (no Tailwind v4 theme rendering issues)

### Dashboard
- [ ] Given a returning user lands on the dashboard, when they view their sites, then no duplicate subscription cards appear
- [ ] Given a user has sites, when they view the dashboard, then each site shows status (generating, live, draft) with quick actions

### Trust & Polish
- [ ] Given a new user arrives, when they see the start page, then they see social proof elements (e.g., "X sites built", trusted-by logos, or testimonials)
- [ ] Given a user is in the onboarding flow, when they look at navigation, then they see a clear progress indicator showing what's ahead (not just dots — labeled sections like "Vision | Design | Content | Launch")
- [ ] Given any page in the flow, when the user views it, then the color scheme is consistent, modern, and renders correctly across Chrome/Firefox/Safari

## Dependencies
- Tailwind v4 theming research must complete before color work (Sprint 23 blocker)
- None for UX research phase

## Technical Notes
- **Tailwind v4 @theme issue**: Sprint 23 proved that `@theme inline` in `globals.css` doesn't reliably emit custom CSS variables. The research phase MUST produce a working spike before any color migration is attempted. See memory: `sprint23_tailwind_issue.md`
- **Split-pane preview**: If adopted, consider a responsive approach — full preview on desktop, collapsible on mobile. May use React state to sync onboarding inputs → preview component.
- **AI insight cards**: These should be lightweight — no additional API calls. Parse existing onboarding state (industry detection, page suggestions, color extraction) and surface what's already computed.
- **Dashboard dedup**: Likely a data/query issue — investigate whether the subscription card component is rendered multiple times or if the API returns duplicate entries.

## Out of Scope
- Changing the 11-step onboarding flow structure (steps stay the same, UX within each step changes)
- Adding new AI capabilities (only surfacing what AI already computes)
- Mobile app or native experiences
- Pricing/billing changes
