# US-079: "Archie" AI Persona Branding & New InferenceCards

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a user interacting with AI-powered steps, I want the AI to have a consistent name ("Archie") and to see what Archie understood on every step where AI processes my input, so that the experience feels personal and I can verify the AI is interpreting my inputs correctly.

## Acceptance Criteria
- [ ] Given the idea step InferenceCard, when it renders, then its title is "Archie understood" instead of "AI understood"
- [ ] Given the audience step InferenceCard, when it renders, then its title is "Archie understood" instead of "AI understood"
- [ ] Given the tone step InferenceCard, when it renders, then its title is "Archie understood" instead of "AI understood"
- [ ] Given the pages step, when AI page suggestions have loaded, then an InferenceCard appears with title "Archie planned your site", showing page count and page titles as a list, with explanation "This structure is based on your industry and what similar businesses need."
- [ ] Given the brand step, when color extraction completes, then an InferenceCard appears with title "Archie extracted your brand", showing primary color and palette count, with explanation "These colors will be applied to buttons, headings, and backgrounds across your site."
- [ ] Given the fonts step, when the user has selected fonts, then an InferenceCard appears with title "How Archie uses your fonts", showing heading and body font selections, with explanation about where each font is applied
- [ ] Given any new InferenceCard, when it renders, then it uses the existing `InferenceCard` component with no structural changes
- [ ] Given any InferenceCard, when the user clicks the edit action, then it scrolls to the relevant input above (not to a separate edit view)

## Dependencies
- None — uses existing InferenceCard component and existing data

## Technical Notes
- Existing InferenceCards: change `title` prop in idea, audience, tone `page.tsx` files
- Pages step: data already available from AI page suggestions state — wire into InferenceCard in `insightSlot`
- Brand step: color extraction data already available in component state — wire into InferenceCard
- Fonts step: static card, no API data needed — show selected font names
- All use `StepLayout`'s `insightSlot` prop for positioning
