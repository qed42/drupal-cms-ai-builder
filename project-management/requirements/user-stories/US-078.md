# US-078: Contextual Navigation Buttons

**Epic:** Transparent AI Onboarding
**Priority:** P1
**Milestone:** M20 — AI Transparency

## User Story
As a user navigating the onboarding wizard, I want the "Continue" button to preview where I'm going next — especially at section boundaries, so that I feel oriented in the journey and understand the flow.

## Acceptance Criteria
- [ ] Given the idea step (last step in "Your Business" section), when the continue button renders, then it reads "Next: Your Audience"
- [ ] Given the audience step (section boundary to "Site Structure"), when the continue button renders, then it reads "Next: Site Structure"
- [ ] Given the brand step (section boundary to "Brand & Style"), when the continue button renders, then it reads "Next: Brand & Style"
- [ ] Given the tone step (section boundary to "Review & Build"), when the continue button renders, then it reads "Next: Review & Build"
- [ ] Given steps within a section (theme, name, pages, design, fonts, follow-up), when the continue button renders, then it reads "Continue"
- [ ] Given the start page, when the CTA button renders, then it reads "Let's Go"
- [ ] Given the review-settings page, when the generate button renders, then it still reads "Generate My Website"

## Dependencies
- None — prop changes only

## Technical Notes
- Each step's `page.tsx` passes `buttonLabel` to `StepLayout`
- Some already use contextual labels; this story formalizes the pattern
- Full button mapping in `architecture-onboarding-ux-polish.md` Section 3
