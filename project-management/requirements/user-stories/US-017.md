# US-017: Structured Content Population

**Epic:** AI Website Generation
**Priority:** P0
**Milestone:** M2 — AI Site Generation Engine

## User Story
As a business owner, I want the AI to populate my content types with realistic sample content, so that my site looks complete and I only need to replace details — not build from scratch.

## Acceptance Criteria
- [ ] Given a "Service" content type exists, when generation runs, then sample service entities are created based on the services the user listed in onboarding
- [ ] Given a "Team Member" content type exists, when generation runs, then 3–5 sample team member entities are created with realistic names, roles, and bios
- [ ] Given industry-specific content types, when generation runs, then they are populated with industry-relevant sample data
- [ ] Given all generated content, when viewed, then it is created in "draft" status (not published)
- [ ] Given sample content, when viewed on the generated pages, then it is contextually coherent (team members match the industry, services match the business type)

## Dependencies
- US-012 (Content Type Auto-Creation)
- US-014 (SEO Content Generation)

## Technical Notes
- FR-210
- Content Agent generates entity content using LLM based on onboarding context
- All content created as Drupal entities via Entity API
- Content should reference each other where appropriate (e.g., services linked to team members)
