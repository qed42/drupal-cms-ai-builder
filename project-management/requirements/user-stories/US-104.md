# US-104: Code Component Provisioning to Drupal Canvas

**Epic:** Code Component Generation
**Priority:** P0
**Milestone:** M26 — Code Component Generation

## User Story
As a user who publishes their site, I want code component sections to be provisioned as real Canvas Code Component config entities in Drupal, so that my site renders correctly and I can edit it in Canvas after launch.

## Acceptance Criteria
- [ ] Given a blueprint with `_codeComponents[]`, when provisioning runs, then each code component is created as a `canvas.js_component.[machineName]` config entity in Drupal
- [ ] Given a provisioned Code Component, when Canvas renders it, then the JSX source compiles and renders correctly with Tailwind styling applied
- [ ] Given a component tree with mixed SDC + Code Component nodes, when Canvas builds the page, then both types render in their correct positions (coexistence)
- [ ] Given Code Component config YAML, when imported via `drush cim`, then all fields are populated correctly: `js.source`, `js.compiled`, `css.source`, `css.compiled`, `props`, `slots`
- [ ] Given a provisioning failure for one Code Component, when the error occurs, then remaining components still provision (partial failure tolerance) and the error is logged
- [ ] Given the provisioned site, when the user opens Canvas editor, then Code Component sections are editable (props, content) alongside SDC sections

## Dependencies
- US-099 (config YAML builder)
- US-102 (pipeline produces blueprint with code components)

- Architecture: `architecture-code-components.md` Section 7 and Spike Results (Section 11)

- SDC and Code Components coexist in the same `canvas.component_tree_node` structure
