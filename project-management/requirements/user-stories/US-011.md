# US-011: Save & Resume Onboarding

**Epic:** Onboarding Wizard
**Priority:** P2
**Milestone:** M1 — Platform Foundation & Onboarding

## User Story
As a user who started onboarding but didn't finish, I want to resume from where I left off, so that I don't have to re-enter my information.

## Acceptance Criteria
- [ ] Given a user has completed Steps 1–3, when they close the browser and return later, then they see Step 4 with previous data intact
- [ ] Given a user logs back in with an incomplete onboarding, when they reach the dashboard, then they see a "Continue Setup" prompt
- [ ] Given saved onboarding data, when the user resumes, then all previously entered data is pre-filled
- [ ] Given a saved onboarding state, when the trial period expires, then incomplete onboarding data is preserved (but user must subscribe to continue)

## Dependencies
- US-005 through US-009 (Onboarding Steps 1–5)

## Technical Notes
- FR-108
- Onboarding state stored on the site profile entity with a "current step" field
- Each step auto-saves on "Next" — this is mostly about detecting incomplete state on login and redirecting
