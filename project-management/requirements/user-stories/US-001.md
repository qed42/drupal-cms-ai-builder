# US-001: User Registration

**Epic:** User Account & Trial Management
**Priority:** P0
**Milestone:** M1 — Platform Foundation & Onboarding

## User Story
As a business owner, I want to register with my email address, so that I can start building my website.

## Acceptance Criteria
- [ ] Given a user visits the platform, when they click "Build My Website", then they see a registration form
- [ ] Given the registration form, when a user enters a valid email and password, then an account is created
- [ ] Given a valid registration, when the account is created, then the user is automatically logged in and redirected to the onboarding wizard
- [ ] Given an existing email, when a user tries to register, then they see an error with a link to login
- [ ] Given a registration, when the account is created, then a welcome/verification email is sent

## Dependencies
- None (foundation story)

## Technical Notes
- Drupal user entity with custom registration form (simplified, not default Drupal registration)
- Email verification flow — decide if blocking or non-blocking (recommend non-blocking for MVP to reduce friction)
- Consider social login (Google) as a Should-have enhancement
