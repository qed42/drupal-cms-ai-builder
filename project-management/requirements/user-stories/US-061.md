# US-061: Vision Deck & Content Comparisons

**Epic:** Vision & Communication
**Priority:** P1
**Milestone:** M10 — Quality Framework
**PRD Refs:** FR-601, FR-602

## User Story
As a stakeholder, I want a presentation deck and before/after content comparisons, so that I can understand the v2 improvements and share them with the team.

## Acceptance Criteria
- [ ] Given the vision deck, when created, then it covers: (a) problem statement (current shallow content), (b) solution overview (Research → Plan → Generate → Review pipeline), (c) before/after demo with 2-3 test cases, (d) architecture diagram, (e) phased roadmap
- [ ] Given the deck format, when produced, then it is in markdown-based slides (Marp or reveal.js) or PDF
- [ ] Given before/after comparisons, when produced for 3 representative test cases (healthcare, legal, restaurant), then they show side-by-side: word count delta, new content sections, improved specificity examples, and SEO improvements

## Dependencies
- US-060 (Content Quality Validation & Comparison Artifacts)

## Technical Notes
- Marp (markdown → slides) is the simplest approach — single `.md` file generates a slide deck
- Before/after comparisons should use actual generated output from the synthetic test cases
- This is a communication artifact, not a software deliverable — lower engineering effort
