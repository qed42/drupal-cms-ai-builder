# US-114: Responsive Preview Toolbar

**Epic:** Live Blueprint Preview
**Priority:** P1
**Milestone:** M28 — Live Blueprint Preview

## User Story
As a user reviewing my generated website, I want to switch between desktop, tablet, and mobile viewport sizes in the preview, so that I can verify my site looks good across different devices.

## Acceptance Criteria
- [ ] Given the live preview is rendered, when the user clicks the desktop viewport button, then the iframe renders at full available width (no constraint)
- [ ] Given the live preview is rendered, when the user clicks the tablet viewport button, then the iframe renders at 768px width centered in the preview area with a visible border showing the device frame
- [ ] Given the live preview is rendered, when the user clicks the mobile viewport button, then the iframe renders at 375px width centered in the preview area with a visible device frame
- [ ] Given the user switches viewport sizes, when the iframe resizes, then Tailwind responsive breakpoints in the code components react correctly (e.g., `md:` classes activate/deactivate)
- [ ] Given the preview toolbar, when the user clicks a zoom control, then the preview scales to 75%, 100%, or 125% while maintaining the effective viewport width for responsive breakpoints
- [ ] Given the current viewport selection, when the user navigates to a different page, then the viewport selection persists

## Dependencies
- US-111 (Live Preview Frame)

## Technical Notes
- Architecture: `architecture-live-blueprint-preview.md` Section 7.1
- New file: `platform-app/src/app/onboarding/review/components/PreviewToolbar.tsx`
- Viewport switching changes the iframe's CSS `width` — this triggers Tailwind responsive breakpoints naturally
- Zoom is implemented via CSS `transform: scale()` on the iframe container with `transform-origin: top center`
- Device frame borders are styled div wrappers around the iframe
- Viewport state is stored in React state (not persisted to server)
